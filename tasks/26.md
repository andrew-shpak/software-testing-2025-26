# Завдання 26: Агрегатор новин

## Предметна область

Платформа агрегації новин зі статтями, джерелами та закладками.

## Сутності

- **Source**: Id, Name, Url, Category (Tech/Sports/Politics/Science/Business), IsActive
- **Article**: Id, SourceId, Title, Summary, Url, PublishedAt, ImageUrl, Category
- **Bookmark**: Id, UserId, ArticleId, CreatedAt, Notes

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/articles | Отримати статті (фільтр за категорією, джерелом, діапазоном дат, пагінація) |
| POST | /api/articles | Створити статтю |
| GET | /api/articles/{id} | Отримати деталі статті |
| GET | /api/articles/trending | Отримати топ статей за кількістю закладок |
| GET | /api/sources | Отримати всі джерела |
| POST | /api/sources | Додати джерело |
| POST | /api/bookmarks | Додати статтю до закладок |
| DELETE | /api/bookmarks/{id} | Видалити закладку |
| GET | /api/bookmarks | Отримати закладки користувача |

## Бізнес-правила

- URL статті має бути унікальним
- URL джерела має бути унікальним
- Не можна додати одну й ту ж статтю до закладок двічі
- Статті неактивних джерел приховані зі списку
- Трендовість визначається за кількістю закладок за останні 7 днів

## Обсяг тестування

- **Модульні тести**: Обчислення трендовості, виявлення дублікатів URL, фільтрація неактивних джерел
- **Інтеграційні тести (WebApplicationFactory)**: Список статей, потік закладок, ендпоінт трендів
- **Тести бази даних (Testcontainers)**: Обмеження унікальності URL, запит трендів із часовим вікном, каскадне видалення джерело-стаття
- **Тести продуктивності (k6)**: Навантажувальне тестування стрічки статей з пагінацією, стрес-тестування обчислення трендів

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Source**: Name
- **Article**: Title, Summary, ImageUrl, PublishedAt
- **Bookmark**: Notes, CreatedAt

> Поля Url (джерело), Url (стаття), Category, IsActive, UserId, ArticleId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
