# Task 26: News Aggregator

## Domain

News aggregation platform with articles, sources, and bookmarks.

## Entities

- **Source**: Id, Name, Url, Category (Tech/Sports/Politics/Science/Business), IsActive
- **Article**: Id, SourceId, Title, Summary, Url, PublishedAt, ImageUrl, Category
- **Bookmark**: Id, UserId, ArticleId, CreatedAt, Notes

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/articles | Get articles (filter by category, source, date range, pagination) |
| POST | /api/articles | Create an article |
| GET | /api/articles/{id} | Get article details |
| GET | /api/articles/trending | Get top articles by bookmark count |
| GET | /api/sources | Get all sources |
| POST | /api/sources | Add a source |
| POST | /api/bookmarks | Bookmark an article |
| DELETE | /api/bookmarks/{id} | Remove bookmark |
| GET | /api/bookmarks | Get user's bookmarked articles |

## Business Rules

- Article URL must be unique
- Source URL must be unique
- Cannot bookmark the same article twice
- Inactive sources' articles are hidden from listing
- Trending is based on bookmark count in last 7 days

## Testing Scope

- **Unit Tests**: Trending calculation, duplicate URL detection, inactive source filtering
- **Integration Tests (WebApplicationFactory)**: Article listing, bookmark flow, trending endpoint
- **Database Tests (Testcontainers)**: URL uniqueness constraints, trending query with date window, source-article cascade
- **Performance Tests (k6)**: Load test article feed with pagination, stress test trending calculation

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Source**: Name
- **Article**: Title, Summary, ImageUrl, PublishedAt
- **Bookmark**: Notes, CreatedAt

> Fields like Url (source), Url (article), Category, IsActive, UserId, ArticleId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
