# Завдання 38: Кур'єрська служба

## Предметна область

Система доставки посилок та відстеження кур'єрів.

## Сутності

- **Package**: Id, SenderId, RecipientName, RecipientAddress, RecipientPhone, Weight, Dimensions, Status (Created/PickedUp/InTransit/OutForDelivery/Delivered/Returned), TrackingCode, CreatedAt
- **Sender**: Id, Name, Email, Phone, Address
- **DeliveryUpdate**: Id, PackageId, Status, Location, Notes, Timestamp, UpdatedBy

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| POST | /api/packages | Створити посилку для доставки |
| GET | /api/packages/{trackingCode} | Відстежити посилку за трекінг-кодом |
| GET | /api/packages/{trackingCode}/updates | Отримати хронологію оновлень доставки |
| POST | /api/packages/{id}/update | Додати оновлення статусу доставки |
| PATCH | /api/packages/{id}/deliver | Позначити як доставлено |
| PATCH | /api/packages/{id}/return | Позначити як повернено |
| GET | /api/senders/{id}/packages | Отримати посилки відправника |
| GET | /api/packages?status={status} | Отримати посилки за статусом |

## Бізнес-правила

- TrackingCode генерується автоматично (унікальний, буквено-цифровий, 12 символів)
- Статус повинен слідувати послідовності: Created -> PickedUp -> InTransit -> OutForDelivery -> Delivered
- Вага повинна бути додатною, максимум 50 кг
- Не можна оновити статус доставлених/повернених посилок
- Кожна зміна статусу створює запис DeliveryUpdate

## Обсяг тестування

- **Модульні тести**: Генерація трекінг-коду, валідація послідовності статусів, обмеження ваги
- **Інтеграційні тести (WebApplicationFactory)**: Життєвий цикл посилки, ендпоінт відстеження, хронологія оновлень доставки
- **Тести бази даних (Testcontainers)**: Унікальність трекінг-коду, аудит оновлень статусу, зв'язок відправник-посилка
- **Тести продуктивності (k6)**: Навантажувальне тестування відстеження посилок, стрес-тестування одночасних оновлень статусу

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Package**: RecipientName, RecipientAddress, RecipientPhone, Dimensions, CreatedAt
- **Sender**: Name, Email, Phone, Address
- **DeliveryUpdate**: Location, Notes, Timestamp, UpdatedBy

> Поля Weight, Status, TrackingCode, SenderId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу повинен відображати реальні співвідношення між батьківськими та дочірніми сутностями.
