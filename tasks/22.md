# Завдання 22: Автосалон

## Предметна область

Управління інвентарем та продажами автосалону.

## Сутності

- **Car**: Id, Make, Model, Year, Color, Mileage, Price, VIN, Status (Available/Reserved/Sold), FuelType (Petrol/Diesel/Electric/Hybrid)
- **Customer**: Id, FirstName, LastName, Email, Phone, DriversLicense
- **Sale**: Id, CarId, CustomerId, SaleDate, SalePrice, PaymentMethod (Cash/Finance/Lease)

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/cars | Отримати автомобілі (фільтр за маркою, діапазоном років, діапазоном цін, типом пального, статусом) |
| POST | /api/cars | Додати автомобіль до інвентарю |
| GET | /api/cars/{id} | Отримати деталі автомобіля |
| PUT | /api/cars/{id} | Оновити інформацію про автомобіль |
| PATCH | /api/cars/{id}/reserve | Зарезервувати автомобіль |
| POST | /api/sales | Зафіксувати продаж |
| GET | /api/sales | Отримати історію продажів (фільтр за діапазоном дат) |
| GET | /api/customers/{id}/purchases | Отримати історію покупок клієнта |

## Бізнес-правила

- VIN має бути унікальним і містити рівно 17 символів
- Лише автомобілі зі статусом Available можуть бути зарезервовані; лише автомобілі зі статусом Available/Reserved можуть бути продані
- SalePrice не може перевищувати вказану Price більш ніж на 5% (буфер для торгу)
- Year має бути між 1900 та поточним роком + 1
- Mileage має бути невід'ємним

## Обсяг тестування

- **Модульні тести**: Валідація VIN, переходи між статусами, валідація ціни продажу
- **Інтеграційні тести (WebApplicationFactory)**: Управління інвентарем, процес резервування, фіксація продажу
- **Тести бази даних (Testcontainers)**: Унікальність VIN, атомарність оновлення статусу, зв'язки продаж-автомобіль-клієнт
- **Тести продуктивності (k6)**: Навантажувальне тестування пошуку автомобілів з кількома фільтрами, стрес-тестування одночасних резервувань

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Car**: Make, Model, Color
- **Customer**: FirstName, LastName, Email, Phone, DriversLicense
- **Sale**: SaleDate

> Поля Year, Mileage, Price, VIN, Status, FuelType, SalePrice, PaymentMethod, CarId, CustomerId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
