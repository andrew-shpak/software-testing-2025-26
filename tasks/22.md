# Task 22: Car Dealership

## Domain

Car dealership inventory and sales management.

## Entities

- **Car**: Id, Make, Model, Year, Color, Mileage, Price, VIN, Status (Available/Reserved/Sold), FuelType (Petrol/Diesel/Electric/Hybrid)
- **Customer**: Id, FirstName, LastName, Email, Phone, DriversLicense
- **Sale**: Id, CarId, CustomerId, SaleDate, SalePrice, PaymentMethod (Cash/Finance/Lease)

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/cars | Get cars (filter by make, year range, price range, fuel type, status) |
| POST | /api/cars | Add a car to inventory |
| GET | /api/cars/{id} | Get car details |
| PUT | /api/cars/{id} | Update car info |
| PATCH | /api/cars/{id}/reserve | Reserve a car |
| POST | /api/sales | Record a sale |
| GET | /api/sales | Get sales history (filter by date range) |
| GET | /api/customers/{id}/purchases | Get purchase history for a customer |

## Business Rules

- VIN must be unique and exactly 17 characters
- Only Available cars can be reserved; only Available/Reserved cars can be sold
- SalePrice cannot exceed listed Price by more than 5% (negotiation buffer)
- Year must be between 1900 and current year + 1
- Mileage must be non-negative

## Testing Scope

- **Unit Tests**: VIN validation, status transitions, sale price validation
- **Integration Tests (WebApplicationFactory)**: Inventory management, reservation flow, sale recording
- **Database Tests (Testcontainers)**: VIN uniqueness, status update atomicity, sale-car-customer relationships
- **Performance Tests (k6)**: Load test car search with multiple filters, stress test concurrent reservations

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Car**: Make, Model, Color
- **Customer**: FirstName, LastName, Email, Phone, DriversLicense
- **Sale**: SaleDate

> Fields like Year, Mileage, Price, VIN, Status, FuelType, SalePrice, PaymentMethod, CarId, CustomerId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
