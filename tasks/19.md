# Завдання 19: Дошка вакансій

## Домен

Платформа оголошень про роботу з компаніями, вакансіями та заявками.

## Сутності

- **Company**: Id, Name, Description, Website, Industry, LogoUrl
- **JobPosting**: Id, CompanyId, Title, Description, Location, SalaryMin, SalaryMax, Type (FullTime/PartTime/Contract), PostedAt, ExpiresAt, IsActive
- **Application**: Id, JobPostingId, ApplicantName, Email, ResumeUrl, CoverLetter, AppliedAt, Status (Pending/Reviewed/Accepted/Rejected)

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/jobs | Отримати активні вакансії (фільтр за місцем, типом, діапазоном зарплати) |
| POST | /api/jobs | Створити вакансію |
| GET | /api/jobs/{id} | Отримати деталі вакансії |
| PUT | /api/jobs/{id} | Оновити вакансію |
| DELETE | /api/jobs/{id} | Закрити вакансію |
| POST | /api/jobs/{id}/apply | Подати заявку |
| GET | /api/jobs/{id}/applications | Отримати заявки на вакансію |
| PATCH | /api/applications/{id}/status | Оновити статус заявки |

## Бізнес-правила

- SalaryMin має бути менше SalaryMax
- Прострочені вакансії автоматично стають неактивними
- Не можна подати заявку на неактивну/прострочену вакансію
- Одна заявка на електронну адресу на вакансію
- ExpiresAt має бути в майбутньому при створенні

## Обсяг тестування

- **Юніт-тести**: Валідація діапазону зарплати, логіка закінчення терміну, перевірка дублікатів заявок
- **Інтеграційні тести (WebApplicationFactory)**: Процес створення вакансії, подання заявки, оновлення статусу, фільтрація прострочених вакансій
- **Тести бази даних (Testcontainers)**: Обмеження унікальності заявки, запити на основі дати закінчення, каскадне видалення компанія-вакансія
- **Тести продуктивності (k6)**: Навантажувальне тестування пошуку вакансій з кількома фільтрами, стрес-тестування подання заявок

## Поля AutoFixture

Поля для автогенерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Company**: Name, Description, Website, Industry, LogoUrl
- **JobPosting**: Title, Description, Location, PostedAt
- **Application**: ApplicantName, ResumeUrl, CoverLetter, AppliedAt

> Поля SalaryMin, SalaryMax, Type, ExpiresAt, IsActive, Email, Status мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як СУБД. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
