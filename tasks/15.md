# Task 15: Weather Station

## Domain

Weather data collection and reporting from multiple stations.

## Entities

- **Station**: Id, Name, Latitude, Longitude, Altitude, IsActive
- **Reading**: Id, StationId, Temperature, Humidity, WindSpeedKmh, Pressure, RecordedAt
- **Alert**: Id, StationId, Type (HighTemp/LowTemp/HighWind/Storm), Message, CreatedAt, IsAcknowledged

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/stations | Get all stations |
| POST | /api/stations | Register a station |
| POST | /api/stations/{id}/readings | Submit a weather reading |
| GET | /api/stations/{id}/readings | Get readings (filter by date range) |
| GET | /api/stations/{id}/latest | Get latest reading |
| GET | /api/stations/{id}/average?from={date}&to={date} | Get average values for period |
| GET | /api/alerts | Get active (unacknowledged) alerts |
| PATCH | /api/alerts/{id}/acknowledge | Acknowledge an alert |

## Business Rules

- Alerts auto-generated when thresholds exceeded (temp > 40°C, wind > 100 km/h)
- Humidity must be 0-100%
- Temperature range: -60°C to 60°C
- Readings must be in chronological order per station
- Inactive stations cannot submit readings

## Testing Scope

- **Unit Tests**: Threshold alert generation, value range validation, average calculation
- **Integration Tests (WebApplicationFactory)**: Reading submission with alert creation, average endpoint, filtering
- **Database Tests (Testcontainers)**: Time-series queries, alert generation on insert trigger, station-reading relationship
- **Performance Tests (k6)**: Load test reading submission (high-frequency data), stress test average calculation over large datasets

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Station**: Name, Latitude, Longitude, Altitude
- **Alert**: Message, CreatedAt
- **Reading**: RecordedAt

> Fields like Temperature, Humidity, WindSpeedKmh, Pressure, IsActive, Type (alert), IsAcknowledged have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
