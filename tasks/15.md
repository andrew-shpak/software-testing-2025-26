# Завдання 15: Метеостанція

## Домен

Збір та звітування метеорологічних даних з кількох станцій.

## Сутності

- **Station**: Id, Name, Latitude, Longitude, Altitude, IsActive
- **Reading**: Id, StationId, Temperature, Humidity, WindSpeedKmh, Pressure, RecordedAt
- **Alert**: Id, StationId, Type (HighTemp/LowTemp/HighWind/Storm), Message, CreatedAt, IsAcknowledged

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/stations | Отримати всі станції |
| POST | /api/stations | Зареєструвати станцію |
| POST | /api/stations/{id}/readings | Подати метеорологічне вимірювання |
| GET | /api/stations/{id}/readings | Отримати вимірювання (фільтр за діапазоном дат) |
| GET | /api/stations/{id}/latest | Отримати останнє вимірювання |
| GET | /api/stations/{id}/average?from={date}&to={date} | Отримати середні значення за період |
| GET | /api/alerts | Отримати активні (непідтверджені) сповіщення |
| PATCH | /api/alerts/{id}/acknowledge | Підтвердити сповіщення |

## Бізнес-правила

- Сповіщення генеруються автоматично при перевищенні порогів (температура > 40°C, вітер > 100 км/год)
- Вологість має бути 0-100%
- Діапазон температур: від -60°C до 60°C
- Вимірювання мають бути в хронологічному порядку для кожної станції
- Неактивні станції не можуть подавати вимірювання

## Обсяг тестування

- **Юніт-тести**: Генерація порогових сповіщень, валідація діапазону значень, розрахунок середнього
- **Інтеграційні тести (WebApplicationFactory)**: Подання вимірювання зі створенням сповіщення, ендпоінт середніх значень, фільтрація
- **Тести бази даних (Testcontainers)**: Запити часових рядів, генерація сповіщень при вставці тригером, зв'язок станція-вимірювання
- **Тести продуктивності (k6)**: Навантажувальне тестування подання вимірювань (високочастотні дані), стрес-тестування розрахунку середніх на великих наборах даних

## Поля AutoFixture

Поля для автогенерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Station**: Name, Latitude, Longitude, Altitude
- **Alert**: Message, CreatedAt
- **Reading**: RecordedAt

> Поля Temperature, Humidity, WindSpeedKmh, Pressure, IsActive, Type (сповіщення), IsAcknowledged мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як СУБД. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
