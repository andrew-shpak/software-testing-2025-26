# Task 9: Music Playlist

## Domain

Music streaming service with songs, albums, and user playlists.

## Entities

- **Song**: Id, Title, Artist, Album, DurationSeconds, Genre, ReleaseDate
- **Playlist**: Id, Name, Description, UserId, CreatedAt, IsPublic
- **PlaylistSong**: Id, PlaylistId, SongId, Position, AddedAt

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/songs | Get all songs (filter by genre, artist) |
| POST | /api/songs | Add a song |
| GET | /api/playlists | Get user playlists |
| POST | /api/playlists | Create a playlist |
| PUT | /api/playlists/{id} | Update playlist details |
| DELETE | /api/playlists/{id} | Delete a playlist |
| POST | /api/playlists/{id}/songs | Add song to playlist |
| DELETE | /api/playlists/{id}/songs/{songId} | Remove song from playlist |
| PUT | /api/playlists/{id}/reorder | Reorder songs in playlist |

## Business Rules

- Song cannot be added to the same playlist twice
- Playlist name must be unique per user
- Positions must be sequential (1, 2, 3...)
- Maximum 100 songs per playlist

## Testing Scope

- **Unit Tests**: Position reordering logic, duplicate song detection, max songs validation
- **Integration Tests (WebApplicationFactory)**: Playlist CRUD, add/remove songs, reorder endpoint
- **Database Tests (Testcontainers)**: Unique song-per-playlist constraint, position integrity after removal, cascade delete
- **Performance Tests (k6)**: Load test song listing with filters, stress test playlist reordering

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Song**: Title, Artist, Album, DurationSeconds, ReleaseDate
- **Playlist**: Name, Description, CreatedAt
- **PlaylistSong**: AddedAt

> Fields like Genre, UserId, IsPublic, Position, PlaylistId, SongId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
