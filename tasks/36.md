# Task 36: Feedback System

## Domain

Product feedback and feature request system with voting and status tracking.

## Entities

- **Feedback**: Id, Title, Description, Type (Bug/Feature/Improvement), Status (Open/UnderReview/Planned/InProgress/Done/Closed), Priority (Low/Medium/High), AuthorName, AuthorEmail, CreatedAt, VoteCount
- **Vote**: Id, FeedbackId, VoterEmail, CreatedAt
- **Comment**: Id, FeedbackId, AuthorName, Content, CreatedAt, IsOfficial

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/feedback | Get all feedback (filter by type, status, sort by votes) |
| POST | /api/feedback | Submit feedback |
| GET | /api/feedback/{id} | Get feedback with comments |
| PUT | /api/feedback/{id} | Update feedback |
| PATCH | /api/feedback/{id}/status | Change status |
| POST | /api/feedback/{id}/vote | Upvote feedback |
| DELETE | /api/feedback/{id}/vote | Remove vote |
| POST | /api/feedback/{id}/comments | Add a comment |
| GET | /api/feedback/stats | Get feedback statistics (by type, by status) |

## Business Rules

- One vote per email per feedback item
- VoteCount is denormalized (updated on vote/unvote)
- Cannot vote on own feedback
- Status transitions must follow workflow (Open → UnderReview → Planned → InProgress → Done)
- Official comments are marked separately

## Testing Scope

- **Unit Tests**: Status transition validation, vote count logic, self-vote prevention
- **Integration Tests (WebApplicationFactory)**: Feedback lifecycle, voting, commenting, status workflow
- **Database Tests (Testcontainers)**: Vote uniqueness, vote count consistency, status transition audit
- **Performance Tests (k6)**: Load test feedback listing with sorting, stress test concurrent voting

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Feedback**: Title, Description, AuthorName, AuthorEmail, CreatedAt
- **Vote**: VoterEmail, CreatedAt
- **Comment**: AuthorName, Content, CreatedAt

> Fields like Type, Status, Priority, VoteCount, FeedbackId, IsOfficial have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
