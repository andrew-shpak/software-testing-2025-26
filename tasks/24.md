# Task 24: Expense Tracker

## Domain

Personal finance tracker for expenses, budgets, and reports.

## Entities

- **Expense**: Id, UserId, Amount, Description, Category, Date, PaymentMethod (Cash/Card/Transfer)
- **Budget**: Id, UserId, Category, MonthlyLimit, Month, Year
- **Category**: Id, Name, Icon, Color

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/expenses | Get expenses (filter by category, date range, payment method) |
| POST | /api/expenses | Add an expense |
| PUT | /api/expenses/{id} | Update an expense |
| DELETE | /api/expenses/{id} | Delete an expense |
| GET | /api/budgets | Get budgets for current month |
| POST | /api/budgets | Set a monthly budget |
| PUT | /api/budgets/{id} | Update a budget |
| GET | /api/reports/monthly?month={m}&year={y} | Get monthly expense report by category |
| GET | /api/reports/summary | Get spending summary (total, average, top category) |

## Business Rules

- Amount must be positive
- Cannot set two budgets for the same category in the same month
- Budget warning when spending reaches 80% of limit
- Budget exceeded flag when spending > limit
- Date cannot be in the future

## Testing Scope

- **Unit Tests**: Budget threshold calculation, report aggregation, date validation
- **Integration Tests (WebApplicationFactory)**: Expense CRUD, budget alerts, monthly report generation
- **Database Tests (Testcontainers)**: Budget uniqueness per category/month, expense aggregation queries, category cascade
- **Performance Tests (k6)**: Load test report generation, stress test bulk expense creation

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Expense**: Description
- **Budget**: CreatedAt
- **Category**: Name, Icon, Color

> Fields like Amount, Category, Date, PaymentMethod, MonthlyLimit, Month, Year, UserId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
