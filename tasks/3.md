# Завдання 3: Менеджер рецептів

## Предметна область

Платформа для обміну рецептами з інгредієнтами та кроками приготування.

## Сутності

- **Recipe**: Id, Title, Description, PrepTimeMinutes, CookTimeMinutes, Servings, Difficulty (Easy/Medium/Hard), CreatedAt
- **Ingredient**: Id, RecipeId, Name, Quantity, Unit
- **Step**: Id, RecipeId, StepNumber, Instruction

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/recipes | Отримати всі рецепти (фільтр за складністю, максимальним часом підготовки) |
| GET | /api/recipes/{id} | Отримати рецепт з інгредієнтами та кроками |
| POST | /api/recipes | Створити рецепт (з інгредієнтами та кроками) |
| PUT | /api/recipes/{id} | Оновити рецепт |
| DELETE | /api/recipes/{id} | Видалити рецепт |
| GET | /api/recipes/search?ingredient={name} | Пошук рецептів за інгредієнтом |

## Бізнес-правила

- Рецепт повинен мати щонайменше один інгредієнт та один крок
- Кроки повинні мати послідовні номери, починаючи з 1
- Час підготовки та час приготування повинні бути додатними
- Кількість порцій має бути щонайменше 1

## Обсяг тестування

- **Модульні тести**: Валідація порядку кроків, валідація часу, логіка пошуку за інгредієнтами
- **Інтеграційні тести (WebApplicationFactory)**: Створення рецепту з вкладеними даними, пошук за інгредієнтом, помилки валідації
- **Тести бази даних (Testcontainers)**: Каскадне видалення (рецепт → інгредієнти + кроки), транзакційне створення рецепту з дочірніми сутностями
- **Тести продуктивності (k6)**: Навантажувальний тест списку рецептів, стрес-тест пошуку за інгредієнтами

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Recipe**: Title, Description, CreatedAt
- **Ingredient**: Name, Unit
- **Step**: Instruction

> Поля PrepTimeMinutes, CookTimeMinutes, Servings, Difficulty, StepNumber, Quantity мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
