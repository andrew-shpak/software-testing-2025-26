# Task 3: Recipe Manager

## Domain

Recipe sharing platform with ingredients and cooking steps.

## Entities

- **Recipe**: Id, Title, Description, PrepTimeMinutes, CookTimeMinutes, Servings, Difficulty (Easy/Medium/Hard), CreatedAt
- **Ingredient**: Id, RecipeId, Name, Quantity, Unit
- **Step**: Id, RecipeId, StepNumber, Instruction

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/recipes | Get all recipes (filter by difficulty, max prep time) |
| GET | /api/recipes/{id} | Get recipe with ingredients and steps |
| POST | /api/recipes | Create a recipe (with ingredients and steps) |
| PUT | /api/recipes/{id} | Update a recipe |
| DELETE | /api/recipes/{id} | Delete a recipe |
| GET | /api/recipes/search?ingredient={name} | Search recipes by ingredient |

## Business Rules

- Recipe must have at least one ingredient and one step
- Steps must have sequential step numbers starting from 1
- PrepTime and CookTime must be positive
- Servings must be at least 1

## Testing Scope

- **Unit Tests**: Step ordering validation, time validation, ingredient search logic
- **Integration Tests (WebApplicationFactory)**: Create recipe with nested data, search by ingredient, validation errors
- **Database Tests (Testcontainers)**: Cascade delete (recipe → ingredients + steps), transactional creation of recipe with children
- **Performance Tests (k6)**: Load test recipe listing, stress test ingredient-based search

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Recipe**: Title, Description, CreatedAt
- **Ingredient**: Name, Unit
- **Step**: Instruction

> Fields like PrepTimeMinutes, CookTimeMinutes, Servings, Difficulty, StepNumber, Quantity have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
