# Завдання 1: Книгарня

## Предметна область

Онлайн-книгарня для управління книгами, авторами та категоріями.

## Сутності

- **Book**: Id, Title, ISBN, Price, PublishedDate, AuthorId, CategoryId
- **Author**: Id, Name, Bio, Country
- **Category**: Id, Name, Description

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/books | Отримати всі книги (з пагінацією, фільтрацією за категорією/автором) |
| GET | /api/books/{id} | Отримати книгу за ID |
| POST | /api/books | Створити нову книгу |
| PUT | /api/books/{id} | Оновити книгу |
| DELETE | /api/books/{id} | Видалити книгу |
| GET | /api/authors | Отримати всіх авторів |
| POST | /api/authors | Створити автора |
| GET | /api/books/search?query={query} | Пошук книг за назвою або ISBN |

## Бізнес-правила

- ISBN має бути унікальним та валідним (13 цифр)
- Ціна має бути додатною
- Книгу не можна створити без існуючого автора
- Видалення автора з пов'язаними книгами має бути заборонено

## Обсяг тестування

- **Модульні тести**: Валідація книги (формат ISBN, діапазон ціни), логіка пошуку, пагінація
- **Інтеграційні тести (WebApplicationFactory)**: Повне тестування CRUD ендпоінтів, фільтрація за параметрами запиту, відповіді з помилками (404, 400, 409)
- **Тести бази даних (Testcontainers)**: Обмеження унікальності ISBN, каскадна поведінка, цілісність зв'язку автор-книга
- **Тести продуктивності (k6)**: Навантажувальний тест GET /api/books з пагінацією, стрес-тест ендпоінту пошуку

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Book**: Title, Description, PublishedDate
- **Author**: Name, Bio, Country
- **Category**: Name, Description

> Поля ISBN, Price, AuthorId, CategoryId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
