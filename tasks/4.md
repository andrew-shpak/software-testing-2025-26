# Task 4: Task Tracker

## Domain

Project task management system with boards and task assignments.

## Entities

- **Board**: Id, Name, Description, CreatedAt
- **TaskItem**: Id, BoardId, Title, Description, Status (Todo/InProgress/Done), Priority (Low/Medium/High/Critical), AssigneeId, DueDate, CreatedAt
- **Assignee**: Id, Name, Email

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/boards | Get all boards |
| POST | /api/boards | Create a board |
| GET | /api/boards/{id}/tasks | Get tasks for a board (filter by status, priority) |
| POST | /api/boards/{id}/tasks | Create a task in a board |
| PUT | /api/tasks/{id} | Update a task |
| PATCH | /api/tasks/{id}/status | Change task status |
| DELETE | /api/tasks/{id} | Delete a task |
| GET | /api/tasks/overdue | Get all overdue tasks |

## Business Rules

- Status transitions: Todo → InProgress → Done (no skipping back)
- DueDate must be in the future when creating a task
- Critical priority tasks must have an assignee
- Board name must be unique

## Testing Scope

- **Unit Tests**: Status transition validation, priority-assignee rule, overdue detection logic
- **Integration Tests (WebApplicationFactory)**: CRUD workflows, status transition endpoint, filtering, overdue endpoint
- **Database Tests (Testcontainers)**: Unique board name constraint, task-board relationship, bulk status updates
- **Performance Tests (k6)**: Load test board task listing with filters, stress test concurrent task creation

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Board**: Name, Description, CreatedAt
- **TaskItem**: Title, Description, CreatedAt
- **Assignee**: Name, Email

> Fields like Status, Priority, DueDate, AssigneeId, BoardId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
