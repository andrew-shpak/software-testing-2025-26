# Завдання 41: Планувальник саду

## Предметна область

Додаток для планування саду з управлінням рослинами, грядками та розкладами поливу.

## Сутності

- **Garden**: Id, Name, Location, SizeSquareMeters, SoilType (Clay/Sandy/Loam/Silt), SunExposure (Full/Partial/Shade)
- **Plant**: Id, Name, Species, WaterFrequencyDays, SunRequirement (Full/Partial/Shade), GrowthDurationDays, PlantingSeason (Spring/Summer/Fall)
- **Planting**: Id, GardenId, PlantId, PlantedDate, ExpectedHarvestDate, Status (Growing/Harvested/Dead), Notes

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/gardens | Отримати всі сади |
| POST | /api/gardens | Створити сад |
| GET | /api/gardens/{id} | Отримати сад з посадками |
| GET | /api/plants | Отримати всі рослини (фільтр за вимогами до сонця, сезоном) |
| POST | /api/plants | Додати рослину |
| POST | /api/gardens/{id}/plantings | Посадити щось у саду |
| PUT | /api/plantings/{id} | Оновити статус посадки |
| GET | /api/gardens/{id}/watering-schedule | Отримати розклад поливу для всіх активних посадок |
| GET | /api/plantings/ready-to-harvest | Отримати посадки, готові до збору |

## Бізнес-правила

- SunRequirement рослини повинен бути сумісним з SunExposure саду
- ExpectedHarvestDate = PlantedDate + GrowthDurationDays
- Розклад поливу формується на основі WaterFrequencyDays кожної рослини
- Не можна садити поза PlantingSeason рослини
- Відстеження простору саду (кожна посадка займає місце залежно від типу рослини)

## Обсяг тестування

- **Модульні тести**: Перевірка сумісності освітлення, розрахунок дати збору, генерація розкладу поливу, валідація сезону
- **Інтеграційні тести (WebApplicationFactory)**: Процес посадки, розклад поливу, готовність до збору
- **Тести бази даних (Testcontainers)**: Зв'язки сад-посадка, запити активних посадок, генерація розкладу
- **Тести продуктивності (k6)**: Навантажувальне тестування генерації розкладу поливу, стрес-тестування масових операцій посадки

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Garden**: Name, Location
- **Plant**: Name, Species
- **Planting**: Notes

> Поля SizeSquareMeters, SoilType, SunExposure, WaterFrequencyDays, SunRequirement, GrowthDurationDays, PlantingSeason, PlantedDate, ExpectedHarvestDate, Status мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу повинен відображати реальні співвідношення між батьківськими та дочірніми сутностями.
