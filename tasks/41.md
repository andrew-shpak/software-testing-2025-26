# Task 41: Garden Planner

## Domain

Garden planning app for managing plants, garden beds, and watering schedules.

## Entities

- **Garden**: Id, Name, Location, SizeSquareMeters, SoilType (Clay/Sandy/Loam/Silt), SunExposure (Full/Partial/Shade)
- **Plant**: Id, Name, Species, WaterFrequencyDays, SunRequirement (Full/Partial/Shade), GrowthDurationDays, PlantingSeason (Spring/Summer/Fall)
- **Planting**: Id, GardenId, PlantId, PlantedDate, ExpectedHarvestDate, Status (Growing/Harvested/Dead), Notes

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/gardens | Get all gardens |
| POST | /api/gardens | Create a garden |
| GET | /api/gardens/{id} | Get garden with plantings |
| GET | /api/plants | Get all plants (filter by sun requirement, season) |
| POST | /api/plants | Add a plant |
| POST | /api/gardens/{id}/plantings | Plant something in a garden |
| PUT | /api/plantings/{id} | Update planting status |
| GET | /api/gardens/{id}/watering-schedule | Get watering schedule for all active plantings |
| GET | /api/plantings/ready-to-harvest | Get plantings ready for harvest |

## Business Rules

- Plant's SunRequirement must be compatible with garden's SunExposure
- ExpectedHarvestDate = PlantedDate + GrowthDurationDays
- Watering schedule derived from each plant's WaterFrequencyDays
- Cannot plant outside the plant's PlantingSeason
- Garden space tracking (each planting takes space based on plant type)

## Testing Scope

- **Unit Tests**: Sun compatibility check, harvest date calculation, watering schedule generation, season validation
- **Integration Tests (WebApplicationFactory)**: Planting flow, watering schedule, harvest readiness
- **Database Tests (Testcontainers)**: Garden-planting relationships, active planting queries, schedule generation
- **Performance Tests (k6)**: Load test watering schedule generation, stress test bulk planting operations

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Garden**: Name, Location
- **Plant**: Name, Species
- **Planting**: Notes

> Fields like SizeSquareMeters, SoilType, SunExposure, WaterFrequencyDays, SunRequirement, GrowthDurationDays, PlantingSeason, PlantedDate, ExpectedHarvestDate, Status have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
