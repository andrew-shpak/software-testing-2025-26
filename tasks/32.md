# Завдання 32: Платформа пожертвувань

## Предметна область

Благодійна платформа для пожертвувань із кампаніями, донатами та відстеженням прогресу.

## Сутності

- **Campaign**: Id, Title, Description, GoalAmount, CurrentAmount, StartDate, EndDate, Status (Active/Completed/Cancelled), OrganizerId
- **Donation**: Id, CampaignId, DonorName, DonorEmail, Amount, Message, CreatedAt, IsAnonymous
- **Organizer**: Id, Name, Email, Organization, IsVerified

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/campaigns | Отримати активні кампанії |
| POST | /api/campaigns | Створити кампанію |
| GET | /api/campaigns/{id} | Отримати кампанію з прогресом |
| PUT | /api/campaigns/{id} | Оновити кампанію |
| POST | /api/campaigns/{id}/donate | Зробити пожертвування |
| GET | /api/campaigns/{id}/donations | Отримати пожертвування (приховати ім'я, якщо анонімне) |
| GET | /api/campaigns/{id}/stats | Отримати статистику кампанії (загальна сума, середнє, кількість донорів) |
| PATCH | /api/campaigns/{id}/close | Закрити кампанію |

## Бізнес-правила

- Сума пожертвування має бути щонайменше $1
- Не можна жертвувати на закриті/скасовані кампанії
- CurrentAmount оновлюється з кожним пожертвуванням
- Кампанія автоматично завершується при досягненні GoalAmount
- Анонімні пожертвування приховують ім'я донора у списках
- Лише верифіковані організатори можуть створювати кампанії

## Обсяг тестування

- **Модульні тести**: Обчислення прогресу, логіка анонімізації, тригер автоматичного завершення
- **Інтеграційні тести (WebApplicationFactory)**: Потік пожертвувань, анонімний список, життєвий цикл кампанії
- **Тести бази даних (Testcontainers)**: Консистентність CurrentAmount при одночасних пожертвуваннях, агрегація статистики, верифікація організатора
- **Тести продуктивності (k6)**: Навантажувальне тестування списку кампаній, стрес-тестування одночасних пожертвувань на популярну кампанію

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Campaign**: Title, Description
- **Donation**: DonorName, DonorEmail, Message, CreatedAt
- **Organizer**: Name, Email, Organization

> Поля GoalAmount, CurrentAmount, StartDate, EndDate, Status, Amount, IsAnonymous, IsVerified, OrganizerId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
