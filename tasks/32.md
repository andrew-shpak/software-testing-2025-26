# Task 32: Donation Platform

## Domain

Charity donation platform with campaigns, donations, and progress tracking.

## Entities

- **Campaign**: Id, Title, Description, GoalAmount, CurrentAmount, StartDate, EndDate, Status (Active/Completed/Cancelled), OrganizerId
- **Donation**: Id, CampaignId, DonorName, DonorEmail, Amount, Message, CreatedAt, IsAnonymous
- **Organizer**: Id, Name, Email, Organization, IsVerified

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/campaigns | Get active campaigns |
| POST | /api/campaigns | Create a campaign |
| GET | /api/campaigns/{id} | Get campaign with progress |
| PUT | /api/campaigns/{id} | Update campaign |
| POST | /api/campaigns/{id}/donate | Make a donation |
| GET | /api/campaigns/{id}/donations | Get donations (hide name if anonymous) |
| GET | /api/campaigns/{id}/stats | Get campaign statistics (total, average, donor count) |
| PATCH | /api/campaigns/{id}/close | Close a campaign |

## Business Rules

- Donation amount must be at least $1
- Cannot donate to closed/cancelled campaigns
- CurrentAmount updated on each donation
- Campaign auto-completes when GoalAmount reached
- Anonymous donations hide donor name in listings
- Only verified organizers can create campaigns

## Testing Scope

- **Unit Tests**: Progress calculation, anonymization logic, auto-completion trigger
- **Integration Tests (WebApplicationFactory)**: Donation flow, anonymous listing, campaign lifecycle
- **Database Tests (Testcontainers)**: CurrentAmount consistency under concurrent donations, stats aggregation, organizer verification
- **Performance Tests (k6)**: Load test campaign listing, stress test concurrent donations to popular campaign

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Campaign**: Title, Description
- **Donation**: DonorName, DonorEmail, Message, CreatedAt
- **Organizer**: Name, Email, Organization

> Fields like GoalAmount, CurrentAmount, StartDate, EndDate, Status, Amount, IsAnonymous, IsVerified, OrganizerId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
