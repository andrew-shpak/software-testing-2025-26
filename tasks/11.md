# Task 11: E-Commerce Cart

## Domain

Shopping cart system with products, cart management, and order placement.

## Entities

- **Product**: Id, Name, Description, Price, StockQuantity, Category, SKU
- **Cart**: Id, UserId, CreatedAt, UpdatedAt
- **CartItem**: Id, CartId, ProductId, Quantity, UnitPrice
- **Order**: Id, UserId, Items (snapshot), TotalAmount, Status (Pending/Paid/Shipped), CreatedAt

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/products | Get products (filter by category, price range) |
| POST | /api/products | Create a product |
| GET | /api/cart | Get current user cart |
| POST | /api/cart/items | Add item to cart |
| PUT | /api/cart/items/{productId} | Update item quantity |
| DELETE | /api/cart/items/{productId} | Remove item from cart |
| POST | /api/cart/checkout | Place order from cart |
| GET | /api/orders | Get user orders |

## Business Rules

- Cannot add more items than available stock
- Checkout decrements stock and clears cart
- SKU must be unique
- Price at checkout is locked (not affected by later price changes)
- Empty cart cannot be checked out

## Testing Scope

- **Unit Tests**: Stock validation, total calculation, price snapshot logic
- **Integration Tests (WebApplicationFactory)**: Add-to-cart flow, checkout process, stock validation errors
- **Database Tests (Testcontainers)**: Stock decrement atomicity, order snapshot integrity, SKU uniqueness
- **Performance Tests (k6)**: Load test product listing, stress test concurrent checkout (race condition on stock)

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Product**: Name, Description, Category
- **Cart**: CreatedAt, UpdatedAt
- **Order**: CreatedAt

> Fields like Price, StockQuantity, SKU, Quantity, UnitPrice, TotalAmount, Status, UserId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
