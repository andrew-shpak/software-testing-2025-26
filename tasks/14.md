# Завдання 14: Фітнес-трекер

## Домен

Додаток для відстеження фітнесу з тренуваннями, вправами та моніторингом прогресу.

## Сутності

- **User**: Id, Name, Email, HeightCm, WeightKg
- **Workout**: Id, UserId, Name, Date, DurationMinutes, CaloriesBurned, Notes
- **Exercise**: Id, WorkoutId, Name, Sets, Reps, WeightKg, DurationSeconds

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/users/{id}/workouts | Отримати тренування користувача (фільтр за діапазоном дат) |
| POST | /api/workouts | Записати тренування |
| GET | /api/workouts/{id} | Отримати тренування з вправами |
| PUT | /api/workouts/{id} | Оновити тренування |
| DELETE | /api/workouts/{id} | Видалити тренування |
| POST | /api/workouts/{id}/exercises | Додати вправу до тренування |
| GET | /api/users/{id}/stats | Отримати статистику користувача (загальна кількість тренувань, середня тривалість, загальні калорії) |
| GET | /api/users/{id}/progress?exercise={name} | Отримати прогрес для конкретної вправи з часом |

## Бізнес-правила

- CaloriesBurned має бути додатним
- Sets та Reps мають бути додатними цілими числами
- Дата тренування не може бути в майбутньому
- Статистика обчислюється з усіх тренувань за заданий період

## Обсяг тестування

- **Юніт-тести**: Розрахунок статистики, валідація калорій, логіка відстеження прогресу
- **Інтеграційні тести (WebApplicationFactory)**: Процес запису тренування, ендпоінт статистики, запити прогресу
- **Тести бази даних (Testcontainers)**: Запити за діапазоном дат, запити агрегації для статистики, каскадне видалення вправ-тренувань
- **Тести продуктивності (k6)**: Навантажувальне тестування розрахунку статистики, стрес-тестування масового запису тренувань

## Поля AutoFixture

Поля для автогенерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **User**: Name, Email
- **Workout**: Name, Notes
- **Exercise**: Name

> Поля HeightCm, WeightKg, DurationMinutes, CaloriesBurned, Date, Sets, Reps, WeightKg, DurationSeconds мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як СУБД. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
