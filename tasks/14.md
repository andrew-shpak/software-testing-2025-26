# Task 14: Fitness Tracker

## Domain

Fitness tracking app for workouts, exercises, and progress monitoring.

## Entities

- **User**: Id, Name, Email, HeightCm, WeightKg
- **Workout**: Id, UserId, Name, Date, DurationMinutes, CaloriesBurned, Notes
- **Exercise**: Id, WorkoutId, Name, Sets, Reps, WeightKg, DurationSeconds

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/users/{id}/workouts | Get user workouts (filter by date range) |
| POST | /api/workouts | Log a workout |
| GET | /api/workouts/{id} | Get workout with exercises |
| PUT | /api/workouts/{id} | Update a workout |
| DELETE | /api/workouts/{id} | Delete a workout |
| POST | /api/workouts/{id}/exercises | Add exercise to workout |
| GET | /api/users/{id}/stats | Get user statistics (total workouts, avg duration, total calories) |
| GET | /api/users/{id}/progress?exercise={name} | Get progress for a specific exercise over time |

## Business Rules

- CaloriesBurned must be positive
- Sets and Reps must be positive integers
- Workout date cannot be in the future
- Statistics are computed from all workouts in a given period

## Testing Scope

- **Unit Tests**: Statistics calculation, calorie validation, progress tracking logic
- **Integration Tests (WebApplicationFactory)**: Workout logging flow, stats endpoint, progress queries
- **Database Tests (Testcontainers)**: Date range queries, aggregation queries for stats, exercise-workout cascade
- **Performance Tests (k6)**: Load test stats calculation, stress test bulk workout logging

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **User**: Name, Email
- **Workout**: Name, Notes
- **Exercise**: Name

> Fields like HeightCm, WeightKg, DurationMinutes, CaloriesBurned, Date, Sets, Reps, WeightKg, DurationSeconds have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
