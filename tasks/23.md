# Task 23: Online Quiz

## Domain

Quiz platform with question banks, quizzes, and scoring.

## Entities

- **Quiz**: Id, Title, Description, TimeLimit (minutes), PassScore (percentage), IsPublished, CreatedAt
- **Question**: Id, QuizId, Text, Type (SingleChoice/MultipleChoice/TrueFalse), Points
- **Answer**: Id, QuestionId, Text, IsCorrect
- **Attempt**: Id, QuizId, UserName, StartedAt, FinishedAt, Score, IsPassed

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/quizzes | Get published quizzes |
| POST | /api/quizzes | Create a quiz |
| GET | /api/quizzes/{id} | Get quiz with questions (no answers if taking) |
| POST | /api/quizzes/{id}/questions | Add a question with answers |
| POST | /api/quizzes/{id}/start | Start a quiz attempt |
| POST | /api/attempts/{id}/submit | Submit answers and get score |
| GET | /api/attempts/{id}/results | Get detailed results |
| GET | /api/quizzes/{id}/leaderboard | Get top scores |

## Business Rules

- Each question must have at least 2 answers
- SingleChoice must have exactly 1 correct answer
- Score = (earned points / total points) × 100
- Attempt must be submitted before time limit expires
- Cannot start a new attempt if one is in progress

## Testing Scope

- **Unit Tests**: Score calculation, answer validation, time limit enforcement
- **Integration Tests (WebApplicationFactory)**: Quiz creation flow, attempt lifecycle, scoring, leaderboard
- **Database Tests (Testcontainers)**: Question-answer integrity, attempt tracking, score aggregation
- **Performance Tests (k6)**: Load test leaderboard queries, stress test concurrent quiz submissions

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Quiz**: Title, Description, CreatedAt
- **Question**: Text
- **Answer**: Text
- **Attempt**: UserName

> Fields like TimeLimit, PassScore, IsPublished, Type, Points, IsCorrect, StartedAt, FinishedAt, Score, IsPassed have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
