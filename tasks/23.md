# Завдання 23: Онлайн-вікторина

## Предметна область

Платформа вікторин з банками питань, вікторинами та підрахунком балів.

## Сутності

- **Quiz**: Id, Title, Description, TimeLimit (хвилини), PassScore (відсотки), IsPublished, CreatedAt
- **Question**: Id, QuizId, Text, Type (SingleChoice/MultipleChoice/TrueFalse), Points
- **Answer**: Id, QuestionId, Text, IsCorrect
- **Attempt**: Id, QuizId, UserName, StartedAt, FinishedAt, Score, IsPassed

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/quizzes | Отримати опубліковані вікторини |
| POST | /api/quizzes | Створити вікторину |
| GET | /api/quizzes/{id} | Отримати вікторину з питаннями (без відповідей під час проходження) |
| POST | /api/quizzes/{id}/questions | Додати питання з відповідями |
| POST | /api/quizzes/{id}/start | Розпочати спробу вікторини |
| POST | /api/attempts/{id}/submit | Надіслати відповіді та отримати бал |
| GET | /api/attempts/{id}/results | Отримати детальні результати |
| GET | /api/quizzes/{id}/leaderboard | Отримати найкращі результати |

## Бізнес-правила

- Кожне питання має містити щонайменше 2 відповіді
- SingleChoice повинен мати рівно 1 правильну відповідь
- Score = (набрані бали / загальна кількість балів) × 100
- Спроба має бути надіслана до завершення часового ліміту
- Не можна розпочати нову спробу, якщо одна вже в процесі

## Обсяг тестування

- **Модульні тести**: Обчислення балів, валідація відповідей, перевірка часового ліміту
- **Інтеграційні тести (WebApplicationFactory)**: Потік створення вікторини, життєвий цикл спроби, підрахунок балів, таблиця лідерів
- **Тести бази даних (Testcontainers)**: Цілісність питань і відповідей, відстеження спроб, агрегація балів
- **Тести продуктивності (k6)**: Навантажувальне тестування запитів таблиці лідерів, стрес-тестування одночасних надсилань вікторин

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Quiz**: Title, Description, CreatedAt
- **Question**: Text
- **Answer**: Text
- **Attempt**: UserName

> Поля TimeLimit, PassScore, IsPublished, Type, Points, IsCorrect, StartedAt, FinishedAt, Score, IsPassed мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
