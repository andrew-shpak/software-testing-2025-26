# Task 42: Pharmacy Store

## Domain

Pharmacy inventory and prescription management system.

## Entities

- **Medicine**: Id, Name, GenericName, Manufacturer, Price, StockQuantity, ExpiryDate, RequiresPrescription, Category (Antibiotic/Painkiller/Vitamin/Cardiac/Other)
- **Prescription**: Id, PatientName, PatientPhone, DoctorName, DoctorLicense, IssuedDate, ExpiresDate, Status (Active/Fulfilled/Expired)
- **PrescriptionItem**: Id, PrescriptionId, MedicineId, Dosage, Quantity, Instructions
- **Sale**: Id, PrescriptionId (nullable), Items, TotalAmount, SoldAt

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/medicines | Get medicines (filter by category, prescription requirement, availability) |
| POST | /api/medicines | Add a medicine |
| PUT | /api/medicines/{id} | Update medicine info |
| GET | /api/medicines/expiring?days={n} | Get medicines expiring within N days |
| GET | /api/medicines/low-stock | Get medicines below minimum stock |
| POST | /api/prescriptions | Create a prescription |
| GET | /api/prescriptions/{id} | Get prescription details |
| POST | /api/sales | Process a sale |
| GET | /api/sales | Get sales history |

## Business Rules

- Prescription-required medicines cannot be sold without valid prescription
- Prescription expires after IssuedDate + 30 days
- Cannot sell expired medicines
- Stock decremented on sale
- Sale with prescription marks it as fulfilled
- DoctorLicense must be verified (non-empty, valid format)
- Low stock threshold: 10 units

## Testing Scope

- **Unit Tests**: Prescription expiry check, stock validation, prescription requirement enforcement
- **Integration Tests (WebApplicationFactory)**: Sale with/without prescription, expiry filtering, low-stock alerts
- **Database Tests (Testcontainers)**: Stock consistency on concurrent sales, prescription-sale linkage, expiry date queries
- **Performance Tests (k6)**: Load test medicine search, stress test concurrent sales (pharmacy rush hour)

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Medicine**: Name, GenericName, Manufacturer
- **Prescription**: PatientName, PatientPhone, DoctorName
- **PrescriptionItem**: Dosage, Instructions
- **Sale**: SoldAt

> Fields like Price, StockQuantity, ExpiryDate, RequiresPrescription, Category, DoctorLicense, IssuedDate, ExpiresDate, Status, Quantity, TotalAmount, PrescriptionId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
