# Task 10: Blog Platform

## Domain

Blogging platform with posts, comments, and tags.

## Entities

- **Post**: Id, Title, Content, AuthorName, Slug, PublishedAt, IsPublished, ViewCount
- **Comment**: Id, PostId, AuthorName, Content, CreatedAt, IsApproved
- **Tag**: Id, Name
- **PostTag**: PostId, TagId

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/posts | Get published posts (with pagination, filter by tag) |
| GET | /api/posts/{slug} | Get post by slug (increments view count) |
| POST | /api/posts | Create a draft post |
| PUT | /api/posts/{id} | Update a post |
| PATCH | /api/posts/{id}/publish | Publish a draft post |
| DELETE | /api/posts/{id} | Delete a post |
| POST | /api/posts/{id}/comments | Add a comment |
| GET | /api/posts/{id}/comments | Get approved comments for a post |
| PATCH | /api/comments/{id}/approve | Approve a comment |

## Business Rules

- Slug must be auto-generated from title and unique
- Only published posts are visible in listing
- View count increments on each GET by slug
- Comments require approval before being visible
- Cannot publish a post with empty content

## Testing Scope

- **Unit Tests**: Slug generation, view count logic, comment approval flow
- **Integration Tests (WebApplicationFactory)**: Draft-to-publish flow, comment moderation, slug-based retrieval
- **Database Tests (Testcontainers)**: Unique slug constraint, view count concurrency, tag many-to-many relationship
- **Performance Tests (k6)**: Load test post listing with pagination, stress test concurrent view count increments

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Post**: Title, Content, AuthorName, PublishedAt
- **Comment**: AuthorName, Content, CreatedAt
- **Tag**: Name

> Fields like Slug, IsPublished, ViewCount, IsApproved, PostId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
