# Task 16: Event Ticketing

## Domain

Event management and ticket sales platform.

## Entities

- **Event**: Id, Title, Description, Venue, Date, StartTime, EndTime, TotalTickets, AvailableTickets, TicketPrice
- **Ticket**: Id, EventId, BuyerName, BuyerEmail, PurchaseDate, TicketCode, IsUsed
- **Venue**: Id, Name, Address, Capacity

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/events | Get upcoming events (filter by date, venue) |
| POST | /api/events | Create an event |
| GET | /api/events/{id} | Get event details with availability |
| PUT | /api/events/{id} | Update an event |
| POST | /api/events/{id}/tickets | Purchase ticket(s) |
| GET | /api/tickets/{code} | Validate ticket by code |
| PATCH | /api/tickets/{code}/use | Mark ticket as used (entry scan) |
| GET | /api/events/{id}/attendees | Get list of ticket holders |

## Business Rules

- Cannot purchase more tickets than available
- TicketCode must be unique (auto-generated UUID)
- Event TotalTickets cannot exceed venue capacity
- Cannot purchase tickets for past events
- Ticket can only be used once

## Testing Scope

- **Unit Tests**: Ticket code generation, availability check, past event validation
- **Integration Tests (WebApplicationFactory)**: Purchase flow, ticket validation, duplicate use prevention
- **Database Tests (Testcontainers)**: Available ticket decrement atomicity, unique ticket code, venue capacity constraint
- **Performance Tests (k6)**: Load test event listing, stress test concurrent ticket purchases (flash sale scenario)

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Event**: Title, Description
- **Ticket**: BuyerName, BuyerEmail, PurchaseDate
- **Venue**: Name, Address

> Fields like Date, StartTime, EndTime, TotalTickets, AvailableTickets, TicketPrice, TicketCode, IsUsed, Capacity have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
