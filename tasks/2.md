# Task 2: Movie Library

## Domain

Movie catalog with ratings and reviews.

## Entities

- **Movie**: Id, Title, Director, Genre, ReleaseYear, DurationMinutes, Rating
- **Review**: Id, MovieId, UserId, Score (1-10), Comment, CreatedAt
- **User**: Id, Username, Email

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/movies | Get all movies (with filtering by genre, year) |
| GET | /api/movies/{id} | Get movie by ID (includes average rating) |
| POST | /api/movies | Create a movie |
| PUT | /api/movies/{id} | Update a movie |
| DELETE | /api/movies/{id} | Delete a movie |
| POST | /api/movies/{id}/reviews | Add a review |
| GET | /api/movies/{id}/reviews | Get reviews for a movie |

## Business Rules

- Score must be between 1 and 10
- A user can only review a movie once
- Average rating is computed from all reviews
- ReleaseYear cannot be in the future

## Testing Scope

- **Unit Tests**: Rating calculation, score validation, duplicate review detection
- **Integration Tests (WebApplicationFactory)**: CRUD operations, review submission, average rating in response
- **Database Tests (Testcontainers)**: Unique constraint (user+movie review), cascading delete of reviews with movie
- **Performance Tests (k6)**: Load test movie listing with filters, concurrent review submissions

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Movie**: Title, Director, DurationMinutes
- **Review**: Comment, CreatedAt
- **User**: Username, Email

> Fields like Score, Genre, ReleaseYear, MovieId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
