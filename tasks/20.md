# Task 20: Real Estate Listings

## Domain

Property listing platform with listings, agents, and inquiries.

## Entities

- **Property**: Id, Title, Description, Address, City, Price, Area (sqm), Bedrooms, Bathrooms, Type (Apartment/House/Commercial), Status (Available/Sold/Rented), AgentId, ListedAt
- **Agent**: Id, FirstName, LastName, Email, Phone, LicenseNumber
- **Inquiry**: Id, PropertyId, Name, Email, Phone, Message, CreatedAt, IsResponded

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/properties | Get properties (filter by city, type, price range, bedrooms) |
| POST | /api/properties | List a property |
| GET | /api/properties/{id} | Get property details |
| PUT | /api/properties/{id} | Update a property |
| PATCH | /api/properties/{id}/status | Change property status |
| POST | /api/properties/{id}/inquiries | Submit an inquiry |
| GET | /api/agents/{id}/properties | Get properties by agent |
| GET | /api/agents/{id}/inquiries | Get inquiries for agent's properties |

## Business Rules

- Price and Area must be positive
- LicenseNumber must be unique
- Cannot inquire about sold/rented properties
- Agent must exist when listing a property
- Bedrooms and Bathrooms must be non-negative integers

## Testing Scope

- **Unit Tests**: Price/area validation, status transition rules, inquiry eligibility
- **Integration Tests (WebApplicationFactory)**: Listing flow, multi-filter search, inquiry submission
- **Database Tests (Testcontainers)**: Complex filter queries, agent-property relationship, inquiry tracking
- **Performance Tests (k6)**: Load test property search with multiple filters, stress test concurrent inquiries

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Property**: Title, Description, Address, City, ListedAt
- **Agent**: FirstName, LastName, Email, Phone
- **Inquiry**: Name, Phone, Message, CreatedAt

> Fields like Price, Area, Bedrooms, Bathrooms, Type, Status, AgentId, LicenseNumber, Email, IsResponded have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
