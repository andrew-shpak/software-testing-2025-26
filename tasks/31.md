# Task 31: Appointment Scheduler

## Domain

Service appointment booking with providers, services, and time slots.

## Entities

- **Provider**: Id, Name, Specialization, Email, WorkingHours (start/end)
- **Service**: Id, Name, DurationMinutes, Price, Description
- **Appointment**: Id, ProviderId, ServiceId, ClientName, ClientEmail, Date, StartTime, EndTime, Status (Booked/Completed/Cancelled/NoShow)

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/providers | Get all providers |
| POST | /api/providers | Register a provider |
| GET | /api/services | Get all services |
| POST | /api/services | Create a service |
| GET | /api/providers/{id}/slots?date={date} | Get available time slots |
| POST | /api/appointments | Book an appointment |
| PATCH | /api/appointments/{id}/cancel | Cancel an appointment |
| PATCH | /api/appointments/{id}/complete | Mark as completed |
| GET | /api/providers/{id}/appointments?date={date} | Get provider's schedule |

## Business Rules

- Appointments cannot overlap for the same provider
- EndTime = StartTime + Service.DurationMinutes
- Appointments must be within provider's working hours
- Cancellation only allowed 2+ hours before start time
- Available slots computed by subtracting booked slots from working hours

## Testing Scope

- **Unit Tests**: Slot availability calculation, overlap detection, cancellation window
- **Integration Tests (WebApplicationFactory)**: Booking flow, slot availability, cancellation rules
- **Database Tests (Testcontainers)**: Overlapping appointment prevention, schedule queries, provider-service relationships
- **Performance Tests (k6)**: Load test slot availability, stress test concurrent booking of same slot
