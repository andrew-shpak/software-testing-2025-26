# Завдання 5: Ветеринарна клініка

## Предметна область

Управління ветеринарною клінікою: тварини, власники та записи на прийом.

## Сутності

- **Owner**: Id, FirstName, LastName, Phone, Email
- **Pet**: Id, Name, Species (Dog/Cat/Bird/Other), Breed, BirthDate, OwnerId
- **Appointment**: Id, PetId, VetName, Date, Time, Reason, Status (Scheduled/Completed/Cancelled)

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/owners | Отримати всіх власників |
| POST | /api/owners | Зареєструвати власника |
| GET | /api/owners/{id}/pets | Отримати тварин власника |
| POST | /api/pets | Зареєструвати тварину |
| PUT | /api/pets/{id} | Оновити інформацію про тварину |
| POST | /api/appointments | Запланувати прийом |
| GET | /api/appointments?date={date} | Отримати записи за датою |
| PATCH | /api/appointments/{id}/cancel | Скасувати запис |

## Бізнес-правила

- Не можна запланувати два прийоми для однієї тварини на одну дату
- Дата прийому має бути в майбутньому
- Тварина повинна належати зареєстрованому власнику
- Скасовані записи не можна перепланувати (потрібно створити новий)

## Обсяг тестування

- **Модульні тести**: Виявлення конфліктів записів, валідація дат, правила скасування
- **Інтеграційні тести (WebApplicationFactory)**: Процес реєстрації власника та тварини, планування записів, обробка помилок конфліктів
- **Тести бази даних (Testcontainers)**: Обмеження унікальності (тварина + дата), каскадний зв'язок власник-тварина, запити записів за датою
- **Тести продуктивності (k6)**: Навантажувальний тест списку записів за датою, конкурентне планування записів

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Owner**: FirstName, LastName, Phone, Email
- **Pet**: Name, Breed
- **Appointment**: Reason, VetName

> Поля Species, BirthDate, Date, Time, Status, PetId, OwnerId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
