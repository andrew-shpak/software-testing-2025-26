# Task 5: Pet Clinic

## Domain

Veterinary clinic management for pets, owners, and appointments.

## Entities

- **Owner**: Id, FirstName, LastName, Phone, Email
- **Pet**: Id, Name, Species (Dog/Cat/Bird/Other), Breed, BirthDate, OwnerId
- **Appointment**: Id, PetId, VetName, Date, Time, Reason, Status (Scheduled/Completed/Cancelled)

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/owners | Get all owners |
| POST | /api/owners | Register an owner |
| GET | /api/owners/{id}/pets | Get pets for an owner |
| POST | /api/pets | Register a pet |
| PUT | /api/pets/{id} | Update pet info |
| POST | /api/appointments | Schedule an appointment |
| GET | /api/appointments?date={date} | Get appointments by date |
| PATCH | /api/appointments/{id}/cancel | Cancel an appointment |

## Business Rules

- Cannot schedule two appointments for the same pet on the same date
- Appointment date must be in the future
- Pet must belong to a registered owner
- Cancelled appointments cannot be rescheduled (create a new one)

## Testing Scope

- **Unit Tests**: Appointment conflict detection, date validation, cancellation rules
- **Integration Tests (WebApplicationFactory)**: Owner-pet registration flow, appointment scheduling, conflict error handling
- **Database Tests (Testcontainers)**: Unique constraint (pet + date), owner-pet cascade, appointment queries by date
- **Performance Tests (k6)**: Load test appointment listing by date, concurrent appointment scheduling

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Owner**: FirstName, LastName, Phone, Email
- **Pet**: Name, Breed
- **Appointment**: Reason, VetName

> Fields like Species, BirthDate, Date, Time, Status, PetId, OwnerId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
