# Завдання 8: Бронювання авіарейсів

## Предметна область

Система бронювання авіаквитків з маршрутами та управлінням пасажирами.

## Сутності

- **Flight**: Id, FlightNumber, Origin, Destination, DepartureTime, ArrivalTime, TotalSeats, AvailableSeats, Price
- **Passenger**: Id, FirstName, LastName, Email, PassportNumber
- **Booking**: Id, FlightId, PassengerId, SeatNumber, BookingDate, Status (Confirmed/Cancelled/Boarded)

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/flights | Отримати рейси (фільтр за пунктом відправлення, призначення, датою) |
| GET | /api/flights/{id} | Отримати деталі рейсу з доступністю місць |
| POST | /api/bookings | Забронювати рейс |
| GET | /api/bookings/{id} | Отримати деталі бронювання |
| DELETE | /api/bookings/{id} | Скасувати бронювання |
| GET | /api/passengers/{id}/bookings | Отримати всі бронювання пасажира |
| PATCH | /api/bookings/{id}/board | Позначити пасажира як такого, що пройшов посадку |

## Бізнес-правила

- Не можна забронювати, якщо немає доступних місць
- Номер місця має бути унікальним для кожного рейсу
- Номер паспорта має бути дійсним та унікальним для кожного пасажира
- Повернення коштів при скасуванні можливе лише більш ніж за 24 години до вильоту
- Посадка неможлива після часу вильоту

## Обсяг тестування

- **Модульні тести**: Логіка доступності місць, правила повернення коштів при скасуванні, валідація часу посадки
- **Інтеграційні тести (WebApplicationFactory)**: Процес бронювання, скасування, обробка конфліктів місць
- **Тести бази даних (Testcontainers)**: Унікальність місця на рейсі, лічильник доступних місць, історія бронювань
- **Тести продуктивності (k6)**: Навантажувальний тест пошуку рейсів, стрес-тест конкурентного бронювання місць

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Flight**: FlightNumber, Origin, Destination
- **Passenger**: FirstName, LastName, Email
- **Booking**: BookingDate

> Поля DepartureTime, ArrivalTime, TotalSeats, AvailableSeats, Price, SeatNumber, PassportNumber, Status мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
