# Task 8: Flight Reservation

## Domain

Airline flight booking system with routes and passenger management.

## Entities

- **Flight**: Id, FlightNumber, Origin, Destination, DepartureTime, ArrivalTime, TotalSeats, AvailableSeats, Price
- **Passenger**: Id, FirstName, LastName, Email, PassportNumber
- **Booking**: Id, FlightId, PassengerId, SeatNumber, BookingDate, Status (Confirmed/Cancelled/Boarded)

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/flights | Get flights (filter by origin, destination, date) |
| GET | /api/flights/{id} | Get flight details with seat availability |
| POST | /api/bookings | Book a flight |
| GET | /api/bookings/{id} | Get booking details |
| DELETE | /api/bookings/{id} | Cancel a booking |
| GET | /api/passengers/{id}/bookings | Get all bookings for a passenger |
| PATCH | /api/bookings/{id}/board | Mark passenger as boarded |

## Business Rules

- Cannot book if no available seats
- Seat number must be unique per flight
- Passport number must be valid and unique per passenger
- Cancellation refunds only if more than 24 hours before departure
- Cannot board after departure time

## Testing Scope

- **Unit Tests**: Seat availability logic, cancellation refund rules, boarding time validation
- **Integration Tests (WebApplicationFactory)**: Booking flow, cancellation, seat conflict handling
- **Database Tests (Testcontainers)**: Unique seat per flight, available seats counter, booking history
- **Performance Tests (k6)**: Load test flight search, stress test concurrent seat bookings

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Flight**: FlightNumber, Origin, Destination
- **Passenger**: FirstName, LastName, Email
- **Booking**: BookingDate

> Fields like DepartureTime, ArrivalTime, TotalSeats, AvailableSeats, Price, SeatNumber, PassportNumber, Status have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
