# Завдання 18: Меню ресторану

## Домен

Управління меню ресторану зі стравами, категоріями та спеціальними пропозиціями дня.

## Сутності

- **Category**: Id, Name, Description, SortOrder
- **Dish**: Id, Name, Description, Price, CategoryId, IsAvailable, Calories, Allergens (список)
- **DailySpecial**: Id, DishId, Date, DiscountPercent, SpecialPrice

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/menu | Отримати повне меню, згруповане за категоріями |
| GET | /api/categories | Отримати всі категорії |
| POST | /api/categories | Створити категорію |
| GET | /api/dishes | Отримати страви (фільтр за категорією, лише доступні, без алергенів) |
| POST | /api/dishes | Створити страву |
| PUT | /api/dishes/{id} | Оновити страву |
| PATCH | /api/dishes/{id}/availability | Перемкнути доступність страви |
| POST | /api/specials | Створити спеціальну пропозицію дня |
| GET | /api/specials/today | Отримати сьогоднішні спеціальні пропозиції |

## Бізнес-правила

- Ціна має бути додатною
- DiscountPercent має бути 1-50%
- SpecialPrice = Price × (1 - DiscountPercent/100)
- Лише одна спеціальна пропозиція на страву на день
- SortOrder категорії визначає порядок відображення меню

## Обсяг тестування

- **Юніт-тести**: Розрахунок спеціальної ціни, валідація знижки, фільтрація за алергенами
- **Інтеграційні тести (WebApplicationFactory)**: Отримання меню, створення спеціальної пропозиції, перемикання доступності
- **Тести бази даних (Testcontainers)**: Унікальність спеціальної пропозиції на страву на день, порядок категорій, зв'язок страва-категорія
- **Тести продуктивності (k6)**: Навантажувальне тестування отримання повного меню, стрес-тестування ендпоінту спеціальних пропозицій дня

## Поля AutoFixture

Поля для автогенерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Category**: Name, Description
- **Dish**: Name, Description, Allergens
- **DailySpecial**: Date

> Поля Price, SortOrder, IsAvailable, Calories, CategoryId, DiscountPercent, SpecialPrice мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як СУБД. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
