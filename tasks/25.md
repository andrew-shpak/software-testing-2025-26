# Task 25: Chat Room

## Domain

Real-time chat application with rooms and message history.

## Entities

- **Room**: Id, Name, Description, CreatedBy, CreatedAt, IsPrivate, MaxMembers
- **Member**: Id, RoomId, UserId, JoinedAt, Role (Admin/Member)
- **Message**: Id, RoomId, UserId, Content, SentAt, IsEdited, EditedAt

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/rooms | Get public rooms |
| POST | /api/rooms | Create a room |
| GET | /api/rooms/{id}/messages | Get messages (with pagination, cursor-based) |
| POST | /api/rooms/{id}/messages | Send a message |
| PUT | /api/messages/{id} | Edit a message |
| DELETE | /api/messages/{id} | Delete a message |
| POST | /api/rooms/{id}/join | Join a room |
| POST | /api/rooms/{id}/leave | Leave a room |
| GET | /api/rooms/{id}/members | Get room members |

## Business Rules

- Only members can send messages to a room
- Only message author can edit/delete their messages
- Admins can delete any message
- MaxMembers limits who can join
- Room creator is automatically an Admin
- Edited messages must show IsEdited = true

## Testing Scope

- **Unit Tests**: Permission checks, member limit validation, edit tracking
- **Integration Tests (WebApplicationFactory)**: Room lifecycle, message CRUD, join/leave flow, permission enforcement
- **Database Tests (Testcontainers)**: Cursor-based pagination, member role management, message ordering
- **Performance Tests (k6)**: Load test message retrieval (pagination), stress test concurrent message sending

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Room**: Name, Description, CreatedAt
- **Member**: JoinedAt
- **Message**: Content, SentAt

> Fields like CreatedBy, IsPrivate, MaxMembers, Role, UserId, RoomId, IsEdited, EditedAt have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
