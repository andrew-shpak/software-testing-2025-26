# Завдання 25: Чат-кімната

## Предметна область

Застосунок для обміну повідомленнями в реальному часі з кімнатами та історією повідомлень.

## Сутності

- **Room**: Id, Name, Description, CreatedBy, CreatedAt, IsPrivate, MaxMembers
- **Member**: Id, RoomId, UserId, JoinedAt, Role (Admin/Member)
- **Message**: Id, RoomId, UserId, Content, SentAt, IsEdited, EditedAt

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/rooms | Отримати публічні кімнати |
| POST | /api/rooms | Створити кімнату |
| GET | /api/rooms/{id}/messages | Отримати повідомлення (з пагінацією на основі курсора) |
| POST | /api/rooms/{id}/messages | Надіслати повідомлення |
| PUT | /api/messages/{id} | Редагувати повідомлення |
| DELETE | /api/messages/{id} | Видалити повідомлення |
| POST | /api/rooms/{id}/join | Приєднатися до кімнати |
| POST | /api/rooms/{id}/leave | Покинути кімнату |
| GET | /api/rooms/{id}/members | Отримати учасників кімнати |

## Бізнес-правила

- Лише учасники можуть надсилати повідомлення в кімнату
- Лише автор повідомлення може редагувати/видаляти свої повідомлення
- Адміністратори можуть видаляти будь-яке повідомлення
- MaxMembers обмежує кількість учасників, які можуть приєднатися
- Творець кімнати автоматично стає адміністратором
- Відредаговані повідомлення повинні мати IsEdited = true

## Обсяг тестування

- **Модульні тести**: Перевірка прав доступу, валідація ліміту учасників, відстеження редагувань
- **Інтеграційні тести (WebApplicationFactory)**: Життєвий цикл кімнати, CRUD повідомлень, потік приєднання/виходу, перевірка прав доступу
- **Тести бази даних (Testcontainers)**: Пагінація на основі курсора, управління ролями учасників, порядок повідомлень
- **Тести продуктивності (k6)**: Навантажувальне тестування отримання повідомлень (пагінація), стрес-тестування одночасного надсилання повідомлень

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Room**: Name, Description, CreatedAt
- **Member**: JoinedAt
- **Message**: Content, SentAt

> Поля CreatedBy, IsPrivate, MaxMembers, Role, UserId, RoomId, IsEdited, EditedAt мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
