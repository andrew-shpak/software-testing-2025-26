# Завдання 30: Управління складом

## Предметна область

Складські операції із зонами, стелажами та виконанням замовлень.

## Сутності

- **Zone**: Id, Name, Type (Storage/Picking/Shipping), Temperature (Ambient/Cold/Frozen)
- **Shelf**: Id, ZoneId, Code, Capacity, CurrentLoad
- **Item**: Id, Name, SKU, Weight, RequiredTemperature, ShelfId, Quantity, ExpiryDate
- **Order**: Id, Items (список), Status (Pending/Picking/Packed/Shipped), CreatedAt, ShippedAt

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/zones | Отримати всі зони із заповненістю |
| POST | /api/zones | Створити зону |
| GET | /api/shelves | Отримати стелажі (фільтр за зоною, доступною місткістю) |
| POST | /api/items | Розмістити товар на стелажі |
| PUT | /api/items/{id}/move | Перемістити товар на інший стелаж |
| GET | /api/items/expiring?days={n} | Отримати товари, що закінчуються протягом N днів |
| POST | /api/orders | Створити замовлення на виконання |
| PATCH | /api/orders/{id}/status | Оновити статус замовлення |

## Бізнес-правила

- RequiredTemperature товару має відповідати Temperature зони
- Стелаж не може перевищувати свою Capacity (за вагою)
- Прострочені товари не можуть бути включені до замовлень
- SKU має бути унікальним
- Товари замовлення мають бути доступні в достатній кількості

## Обсяг тестування

- **Модульні тести**: Відповідність температур, обчислення місткості, валідація терміну придатності
- **Інтеграційні тести (WebApplicationFactory)**: Розміщення товарів, створення замовлень, сповіщення про закінчення терміну
- **Тести бази даних (Testcontainers)**: Відстеження місткості, обмеження температури, зменшення запасів при виконанні замовлення
- **Тести продуктивності (k6)**: Навантажувальне тестування запитів стелажів, стрес-тестування одночасного виконання замовлень

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Zone**: Name
- **Shelf**: Code
- **Item**: Name
- **Order**: CreatedAt

> Поля Type, Temperature, Capacity, CurrentLoad, SKU, Weight, RequiredTemperature, Quantity, ExpiryDate, ShelfId, Status, Items мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
