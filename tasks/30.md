# Task 30: Warehouse Management

## Domain

Warehouse operations with zones, shelves, and order fulfillment.

## Entities

- **Zone**: Id, Name, Type (Storage/Picking/Shipping), Temperature (Ambient/Cold/Frozen)
- **Shelf**: Id, ZoneId, Code, Capacity, CurrentLoad
- **Item**: Id, Name, SKU, Weight, RequiredTemperature, ShelfId, Quantity, ExpiryDate
- **Order**: Id, Items (list), Status (Pending/Picking/Packed/Shipped), CreatedAt, ShippedAt

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/zones | Get all zones with occupancy |
| POST | /api/zones | Create a zone |
| GET | /api/shelves | Get shelves (filter by zone, available capacity) |
| POST | /api/items | Place item on shelf |
| PUT | /api/items/{id}/move | Move item to different shelf |
| GET | /api/items/expiring?days={n} | Get items expiring within N days |
| POST | /api/orders | Create a fulfillment order |
| PATCH | /api/orders/{id}/status | Update order status |

## Business Rules

- Item's RequiredTemperature must match zone Temperature
- Shelf cannot exceed its Capacity (by weight)
- Expired items cannot be included in orders
- SKU must be unique
- Order items must be available in sufficient quantity

## Testing Scope

- **Unit Tests**: Temperature matching, capacity calculation, expiry validation
- **Integration Tests (WebApplicationFactory)**: Item placement, order creation, expiry alerts
- **Database Tests (Testcontainers)**: Capacity tracking, temperature constraint, order fulfillment inventory deduction
- **Performance Tests (k6)**: Load test shelf queries, stress test concurrent order fulfillment

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Zone**: Name
- **Shelf**: Code
- **Item**: Name
- **Order**: CreatedAt

> Fields like Type, Temperature, Capacity, CurrentLoad, SKU, Weight, RequiredTemperature, Quantity, ExpiryDate, ShelfId, Status, Items have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
