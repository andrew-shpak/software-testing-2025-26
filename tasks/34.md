# Task 34: Loyalty Program

## Domain

Customer loyalty rewards system with points, rewards, and redemptions.

## Entities

- **Customer**: Id, Name, Email, Phone, TierLevel (Bronze/Silver/Gold/Platinum), TotalPoints, JoinDate
- **PointTransaction**: Id, CustomerId, Points, Type (Earned/Redeemed/Expired/Bonus), Description, CreatedAt
- **Reward**: Id, Name, Description, PointsCost, Category, StockQuantity, IsActive

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/customers/{id} | Get customer with points balance |
| POST | /api/customers | Register a customer |
| POST | /api/customers/{id}/earn | Add earned points |
| POST | /api/customers/{id}/redeem | Redeem points for a reward |
| GET | /api/customers/{id}/history | Get point transaction history |
| GET | /api/rewards | Get available rewards |
| POST | /api/rewards | Create a reward |
| GET | /api/customers/{id}/tier | Get tier status and progress |

## Business Rules

- Tier based on total earned points: Bronze(0), Silver(1000), Gold(5000), Platinum(10000)
- Cannot redeem more points than available balance
- Reward must be in stock to redeem
- Points expire after 12 months if unused
- Gold+ members earn 1.5x points; Platinum earns 2x

## Testing Scope

- **Unit Tests**: Tier calculation, point multiplier logic, expiration rules
- **Integration Tests (WebApplicationFactory)**: Earn/redeem flow, tier promotion, reward redemption
- **Database Tests (Testcontainers)**: Point balance consistency, transaction history, reward stock decrement
- **Performance Tests (k6)**: Load test point balance queries, stress test concurrent redemptions

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
