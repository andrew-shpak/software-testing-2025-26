# Task 12: Inventory System

## Domain

Warehouse inventory management with products, stock movements, and suppliers.

## Entities

- **Product**: Id, Name, SKU, Category, MinStockLevel, CurrentStock, Unit
- **StockMovement**: Id, ProductId, Type (In/Out/Adjustment), Quantity, Reason, Timestamp, PerformedBy
- **Supplier**: Id, Name, ContactEmail, Phone

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/products | Get all products (filter by category) |
| POST | /api/products | Create a product |
| PUT | /api/products/{id} | Update product info |
| POST | /api/products/{id}/stock-in | Record stock incoming |
| POST | /api/products/{id}/stock-out | Record stock outgoing |
| GET | /api/products/{id}/movements | Get stock movement history |
| GET | /api/products/low-stock | Get products below minimum stock level |
| GET | /api/suppliers | Get all suppliers |
| POST | /api/suppliers | Add a supplier |

## Business Rules

- Stock-out cannot exceed current stock
- CurrentStock is updated automatically on each movement
- SKU must be unique
- MinStockLevel must be non-negative
- Low-stock alert when CurrentStock < MinStockLevel

## Testing Scope

- **Unit Tests**: Stock calculation, low-stock detection, movement validation
- **Integration Tests (WebApplicationFactory)**: Stock-in/out flow, low-stock endpoint, movement history
- **Database Tests (Testcontainers)**: Stock consistency after movements, concurrent stock-out prevention, movement audit trail
- **Performance Tests (k6)**: Load test product listing, stress test rapid stock movements

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Product**: Name, Category, Unit
- **StockMovement**: Reason, Timestamp, PerformedBy
- **Supplier**: Name, ContactEmail, Phone

> Fields like SKU, MinStockLevel, CurrentStock, Type (In/Out), Quantity, ProductId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
