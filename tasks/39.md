# Task 39: Voting System

## Domain

Online voting platform for polls and elections with result tallying.

## Entities

- **Election**: Id, Title, Description, StartDate, EndDate, Status (Draft/Active/Closed), Type (SingleChoice/RankedChoice)
- **Candidate**: Id, ElectionId, Name, Description, Party, PhotoUrl
- **Vote**: Id, ElectionId, VoterEmail, CandidateId, Rank (for ranked choice), CastAt

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/elections | Get elections (filter by status) |
| POST | /api/elections | Create an election |
| GET | /api/elections/{id} | Get election with candidates |
| POST | /api/elections/{id}/candidates | Add a candidate |
| POST | /api/elections/{id}/vote | Cast a vote |
| GET | /api/elections/{id}/results | Get election results |
| PATCH | /api/elections/{id}/close | Close an election |
| GET | /api/elections/{id}/turnout | Get voter turnout statistics |

## Business Rules

- One vote per email per election
- Can only vote during active election period
- Election must have at least 2 candidates
- Results only visible after election is closed
- RankedChoice: voter must rank all candidates
- SingleChoice: voter selects exactly one candidate

## Testing Scope

- **Unit Tests**: Vote counting, ranked-choice tallying, turnout calculation, voting window
- **Integration Tests (WebApplicationFactory)**: Voting flow, result visibility rules, election lifecycle
- **Database Tests (Testcontainers)**: Vote uniqueness per voter, result aggregation queries, candidate-election cascade
- **Performance Tests (k6)**: Load test result tallying, stress test concurrent voting

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Election**: Title, Description
- **Candidate**: Name, Description, Party, PhotoUrl
- **Vote**: CastAt

> Fields like StartDate, EndDate, Status, Type, VoterEmail, CandidateId, Rank, ElectionId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
