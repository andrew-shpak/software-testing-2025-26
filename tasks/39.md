# Завдання 39: Система голосування

## Предметна область

Онлайн-платформа для голосування в опитуваннях та виборах з підрахунком результатів.

## Сутності

- **Election**: Id, Title, Description, StartDate, EndDate, Status (Draft/Active/Closed), Type (SingleChoice/RankedChoice)
- **Candidate**: Id, ElectionId, Name, Description, Party, PhotoUrl
- **Vote**: Id, ElectionId, VoterEmail, CandidateId, Rank (для рейтингового голосування), CastAt

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/elections | Отримати вибори (фільтр за статусом) |
| POST | /api/elections | Створити вибори |
| GET | /api/elections/{id} | Отримати вибори з кандидатами |
| POST | /api/elections/{id}/candidates | Додати кандидата |
| POST | /api/elections/{id}/vote | Проголосувати |
| GET | /api/elections/{id}/results | Отримати результати виборів |
| PATCH | /api/elections/{id}/close | Закрити вибори |
| GET | /api/elections/{id}/turnout | Отримати статистику явки виборців |

## Бізнес-правила

- Один голос на email на вибори
- Голосувати можна тільки під час активного періоду виборів
- Вибори повинні мати щонайменше 2 кандидатів
- Результати видимі тільки після закриття виборів
- RankedChoice: виборець повинен проранжувати всіх кандидатів
- SingleChoice: виборець обирає рівно одного кандидата

## Обсяг тестування

- **Модульні тести**: Підрахунок голосів, підрахунок рейтингового голосування, розрахунок явки, вікно голосування
- **Інтеграційні тести (WebApplicationFactory)**: Процес голосування, правила видимості результатів, життєвий цикл виборів
- **Тести бази даних (Testcontainers)**: Унікальність голосу на виборця, запити агрегації результатів, каскад кандидат-вибори
- **Тести продуктивності (k6)**: Навантажувальне тестування підрахунку результатів, стрес-тестування одночасного голосування

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Election**: Title, Description
- **Candidate**: Name, Description, Party, PhotoUrl
- **Vote**: CastAt

> Поля StartDate, EndDate, Status, Type, VoterEmail, CandidateId, Rank, ElectionId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу повинен відображати реальні співвідношення між батьківськими та дочірніми сутностями.
