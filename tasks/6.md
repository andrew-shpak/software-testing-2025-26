# Завдання 6: Парковка

## Предметна область

Управління парковкою з відстеженням в'їзду/виїзду транспортних засобів та виставленням рахунків.

## Сутності

- **ParkingLot**: Id, Name, TotalSpaces, HourlyRate, Address
- **ParkingTicket**: Id, ParkingLotId, LicensePlate, VehicleType (Car/Motorcycle/Truck), EntryTime, ExitTime, AmountCharged, IsPaid
- **ParkingSpace**: Id, ParkingLotId, SpaceNumber, IsOccupied, VehicleType

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/lots | Отримати всі парковки з доступністю |
| GET | /api/lots/{id} | Отримати деталі парковки з кількістю місць |
| POST | /api/lots/{id}/entry | Зареєструвати в'їзд транспортного засобу (повертає квиток) |
| POST | /api/tickets/{id}/exit | Зареєструвати виїзд транспортного засобу (обчислює вартість) |
| POST | /api/tickets/{id}/pay | Оплатити квиток |
| GET | /api/lots/{id}/history | Отримати історію в'їздів/виїздів |

## Бізнес-правила

- Не можна в'їхати, якщо немає доступних місць для даного типу транспортного засобу
- Вартість = години паркування × погодинна ставка (мінімум 1 година)
- Вантажівки займають 2 місця
- Транспортний засіб з неоплаченим квитком не може повторно в'їхати
- Номерний знак має бути унікальним для активного квитка (без дублікатів)

## Обсяг тестування

- **Модульні тести**: Обчислення вартості, логіка доступності місць, правило подвійного місця для вантажівок
- **Інтеграційні тести (WebApplicationFactory)**: Процес в'їзду/виїзду, відмова при повній парковці, обробка оплати
- **Тести бази даних (Testcontainers)**: Відстеження зайнятості місць, обробка конкурентних в'їздів, записи виставлення рахунків
- **Тести продуктивності (k6)**: Навантажувальний тест в'їзду/виїзду при високому трафіку, стрес-тест конкурентних в'їздів на майже заповнену парковку

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **ParkingLot**: Name, Address
- **ParkingTicket**: LicensePlate, EntryTime
- **ParkingSpace**: SpaceNumber

> Поля TotalSpaces, HourlyRate, VehicleType, ExitTime, AmountCharged, IsPaid, IsOccupied мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
