# Task 6: Parking Lot

## Domain

Parking lot management with vehicle entry/exit tracking and billing.

## Entities

- **ParkingLot**: Id, Name, TotalSpaces, HourlyRate, Address
- **ParkingTicket**: Id, ParkingLotId, LicensePlate, VehicleType (Car/Motorcycle/Truck), EntryTime, ExitTime, AmountCharged, IsPaid
- **ParkingSpace**: Id, ParkingLotId, SpaceNumber, IsOccupied, VehicleType

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/lots | Get all parking lots with availability |
| GET | /api/lots/{id} | Get lot details with space counts |
| POST | /api/lots/{id}/entry | Register vehicle entry (returns ticket) |
| POST | /api/tickets/{id}/exit | Register vehicle exit (calculates charge) |
| POST | /api/tickets/{id}/pay | Pay for a ticket |
| GET | /api/lots/{id}/history | Get entry/exit history |

## Business Rules

- Cannot enter if no spaces available for vehicle type
- Charge = hours parked × hourly rate (minimum 1 hour)
- Trucks take 2 spaces
- Vehicle with unpaid ticket cannot re-enter
- License plate must be unique per active ticket (no duplicate entries)

## Testing Scope

- **Unit Tests**: Charge calculation, space availability logic, truck double-space rule
- **Integration Tests (WebApplicationFactory)**: Entry/exit flow, full-lot rejection, payment processing
- **Database Tests (Testcontainers)**: Space occupancy tracking, concurrent entry handling, billing records
- **Performance Tests (k6)**: Load test entry/exit under high traffic, stress test concurrent entries to near-full lot

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **ParkingLot**: Name, Address
- **ParkingTicket**: LicensePlate, EntryTime
- **ParkingSpace**: SpaceNumber

> Fields like TotalSpaces, HourlyRate, VehicleType, ExitTime, AmountCharged, IsPaid, IsOccupied have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
