# Task 7: Hotel Booking

## Domain

Hotel room reservation system with room types and guest management.

## Entities

- **Room**: Id, Number, Type (Single/Double/Suite), PricePerNight, Floor, IsAvailable
- **Guest**: Id, FirstName, LastName, Email, Phone, PassportNumber
- **Reservation**: Id, RoomId, GuestId, CheckInDate, CheckOutDate, TotalPrice, Status (Confirmed/CheckedIn/CheckedOut/Cancelled)

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/rooms | Get all rooms (filter by type, availability, date range) |
| GET | /api/rooms/available?checkIn={date}&checkOut={date} | Get available rooms for dates |
| POST | /api/reservations | Create a reservation |
| GET | /api/reservations/{id} | Get reservation details |
| PATCH | /api/reservations/{id}/checkin | Check in a guest |
| PATCH | /api/reservations/{id}/checkout | Check out a guest |
| DELETE | /api/reservations/{id} | Cancel a reservation |

## Business Rules

- Room cannot be double-booked for overlapping dates
- CheckOutDate must be after CheckInDate
- TotalPrice = nights × PricePerNight
- Check-in only allowed on the CheckInDate
- Cancellation not allowed after check-in

## Testing Scope

- **Unit Tests**: Date overlap detection, price calculation, check-in/checkout rules
- **Integration Tests (WebApplicationFactory)**: Reservation flow, availability endpoint, double-booking prevention
- **Database Tests (Testcontainers)**: Overlapping reservation constraint, guest-reservation history, room status tracking
- **Performance Tests (k6)**: Load test availability search, stress test concurrent booking attempts

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Room**: Number, Floor
- **Guest**: FirstName, LastName, Email, Phone, PassportNumber
- **Reservation**: CreatedAt

> Fields like Type, PricePerNight, IsAvailable, CheckInDate, CheckOutDate, TotalPrice, Status have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
