# Task 17: Library Management

## Domain

Public library system for managing books, members, and loans.

## Entities

- **Book**: Id, Title, Author, ISBN, TotalCopies, AvailableCopies, Category
- **Member**: Id, FirstName, LastName, Email, MembershipDate, IsActive
- **Loan**: Id, BookId, MemberId, BorrowDate, DueDate, ReturnDate, Fine

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/books | Get all books (filter by category, availability) |
| POST | /api/books | Add a book |
| GET | /api/members | Get all members |
| POST | /api/members | Register a member |
| POST | /api/loans | Borrow a book |
| POST | /api/loans/{id}/return | Return a book |
| GET | /api/members/{id}/loans | Get loan history for a member |
| GET | /api/loans/overdue | Get all overdue loans |

## Business Rules

- Cannot borrow if no available copies
- Loan period is 14 days; fine of $0.50/day after due date
- Member can borrow max 5 books at a time
- Inactive members cannot borrow
- Returning a book updates AvailableCopies and calculates fine

## Testing Scope

- **Unit Tests**: Fine calculation, borrow limit check, loan period validation
- **Integration Tests (WebApplicationFactory)**: Borrow/return flow, overdue listing, member limit enforcement
- **Database Tests (Testcontainers)**: Available copies consistency, concurrent borrow handling, fine calculation on return
- **Performance Tests (k6)**: Load test book search, stress test concurrent borrows on popular books

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Book**: Title, Author, Category
- **Member**: FirstName, LastName, Email, MembershipDate
- **Loan**: BorrowDate

> Fields like ISBN, TotalCopies, AvailableCopies, IsActive, DueDate, ReturnDate, Fine have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
