# Завдання 17: Управління бібліотекою

## Домен

Система публічної бібліотеки для управління книгами, читачами та видачами.

## Сутності

- **Book**: Id, Title, Author, ISBN, TotalCopies, AvailableCopies, Category
- **Member**: Id, FirstName, LastName, Email, MembershipDate, IsActive
- **Loan**: Id, BookId, MemberId, BorrowDate, DueDate, ReturnDate, Fine

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/books | Отримати всі книги (фільтр за категорією, доступністю) |
| POST | /api/books | Додати книгу |
| GET | /api/members | Отримати всіх читачів |
| POST | /api/members | Зареєструвати читача |
| POST | /api/loans | Взяти книгу |
| POST | /api/loans/{id}/return | Повернути книгу |
| GET | /api/members/{id}/loans | Отримати історію видач читача |
| GET | /api/loans/overdue | Отримати всі прострочені видачі |

## Бізнес-правила

- Не можна взяти книгу, якщо немає доступних примірників
- Термін видачі — 14 днів; штраф $0.50/день після дати повернення
- Читач може мати максимум 5 книг одночасно
- Неактивні читачі не можуть брати книги
- Повернення книги оновлює AvailableCopies та розраховує штраф

## Обсяг тестування

- **Юніт-тести**: Розрахунок штрафу, перевірка ліміту видач, валідація терміну видачі
- **Інтеграційні тести (WebApplicationFactory)**: Процес видачі/повернення, список прострочених, контроль ліміту читача
- **Тести бази даних (Testcontainers)**: Узгодженість доступних примірників, обробка одночасних видач, розрахунок штрафу при поверненні
- **Тести продуктивності (k6)**: Навантажувальне тестування пошуку книг, стрес-тестування одночасних видач популярних книг

## Поля AutoFixture

Поля для автогенерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Book**: Title, Author, Category
- **Member**: FirstName, LastName, Email, MembershipDate
- **Loan**: BorrowDate

> Поля ISBN, TotalCopies, AvailableCopies, IsActive, DueDate, ReturnDate, Fine мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як СУБД. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
