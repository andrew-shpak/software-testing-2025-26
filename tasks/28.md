# Завдання 28: Аукціонна платформа

## Предметна область

Система онлайн-аукціонів з лотами, ставками та управлінням аукціонами.

## Сутності

- **Auction**: Id, Title, Description, StartingPrice, CurrentPrice, StartTime, EndTime, Status (Upcoming/Active/Ended), SellerId, WinnerId
- **Bid**: Id, AuctionId, BidderId, Amount, PlacedAt
- **User**: Id, Username, Email, Balance

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/auctions | Отримати аукціони (фільтр за статусом) |
| POST | /api/auctions | Створити аукціон |
| GET | /api/auctions/{id} | Отримати аукціон з історією ставок |
| POST | /api/auctions/{id}/bids | Зробити ставку |
| GET | /api/auctions/{id}/bids | Отримати ставки аукціону |
| PATCH | /api/auctions/{id}/close | Закрити аукціон (визначити переможця) |
| GET | /api/users/{id}/bids | Отримати історію ставок користувача |
| GET | /api/users/{id}/won | Отримати аукціони, виграні користувачем |

## Бізнес-правила

- Ставка має бути вищою за поточну ціну
- Не можна робити ставку на власний аукціон
- Не можна робити ставку після завершення аукціону
- Переможець — учасник з найвищою ставкою на момент закриття аукціону
- StartingPrice має бути додатним
- EndTime має бути після StartTime

## Обсяг тестування

- **Модульні тести**: Валідація ставок, визначення переможця, переходи статусу на основі часу
- **Інтеграційні тести (WebApplicationFactory)**: Життєвий цикл аукціону, розміщення ставок, закриття та призначення переможця
- **Тести бази даних (Testcontainers)**: Порядок ставок, обробка одночасних ставок, переходи статусу аукціону
- **Тести продуктивності (k6)**: Навантажувальне тестування списку аукціонів, стрес-тестування одночасних ставок (битва ставок в останню хвилину)

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Auction**: Title, Description
- **Bid**: PlacedAt
- **User**: Username, Email

> Поля StartingPrice, CurrentPrice, StartTime, EndTime, Status, SellerId, WinnerId, Amount, BidderId, Balance мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних має бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
