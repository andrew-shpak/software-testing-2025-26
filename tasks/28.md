# Task 28: Auction Platform

## Domain

Online auction system with items, bids, and auction management.

## Entities

- **Auction**: Id, Title, Description, StartingPrice, CurrentPrice, StartTime, EndTime, Status (Upcoming/Active/Ended), SellerId, WinnerId
- **Bid**: Id, AuctionId, BidderId, Amount, PlacedAt
- **User**: Id, Username, Email, Balance

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/auctions | Get auctions (filter by status) |
| POST | /api/auctions | Create an auction |
| GET | /api/auctions/{id} | Get auction with bid history |
| POST | /api/auctions/{id}/bids | Place a bid |
| GET | /api/auctions/{id}/bids | Get bids for auction |
| PATCH | /api/auctions/{id}/close | Close an auction (determine winner) |
| GET | /api/users/{id}/bids | Get user's bid history |
| GET | /api/users/{id}/won | Get auctions won by user |

## Business Rules

- Bid must be higher than current price
- Cannot bid on own auction
- Cannot bid after auction ends
- Winner is the highest bidder when auction closes
- StartingPrice must be positive
- EndTime must be after StartTime

## Testing Scope

- **Unit Tests**: Bid validation, winner determination, time-based status transitions
- **Integration Tests (WebApplicationFactory)**: Auction lifecycle, bid placement, closing and winner assignment
- **Database Tests (Testcontainers)**: Bid ordering, concurrent bid handling, auction status transitions
- **Performance Tests (k6)**: Load test auction listing, stress test concurrent bidding (last-minute bidding war)

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Auction**: Title, Description
- **Bid**: PlacedAt
- **User**: Username, Email

> Fields like StartingPrice, CurrentPrice, StartTime, EndTime, Status, SellerId, WinnerId, Amount, BidderId, Balance have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
