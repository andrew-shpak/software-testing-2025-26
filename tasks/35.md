# Task 35: Document Manager

## Domain

Document management system with folders, files, and version history.

## Entities

- **Folder**: Id, Name, ParentFolderId (nullable), CreatedBy, CreatedAt
- **Document**: Id, FolderId, Name, ContentType, SizeBytes, Version, CreatedAt, UpdatedAt
- **DocumentVersion**: Id, DocumentId, VersionNumber, ChangeSummary, CreatedBy, CreatedAt

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/folders | Get root folders |
| GET | /api/folders/{id} | Get folder contents (subfolders + documents) |
| POST | /api/folders | Create a folder |
| DELETE | /api/folders/{id} | Delete a folder (must be empty) |
| POST | /api/documents | Upload a document |
| GET | /api/documents/{id} | Get document metadata |
| PUT | /api/documents/{id} | Update document (creates new version) |
| DELETE | /api/documents/{id} | Delete a document |
| GET | /api/documents/{id}/versions | Get version history |
| GET | /api/documents/search?name={query} | Search documents by name |

## Business Rules

- Folder name must be unique within parent folder
- Document name must be unique within folder
- Cannot delete non-empty folders
- Each update increments version and creates a version record
- Maximum file size: 50MB
- Nested folder depth maximum: 10 levels

## Testing Scope

- **Unit Tests**: Version increment logic, depth validation, size limit check
- **Integration Tests (WebApplicationFactory)**: Upload/update flow, folder navigation, version history, search
- **Database Tests (Testcontainers)**: Unique name constraints, recursive folder queries, version chain integrity
- **Performance Tests (k6)**: Load test folder navigation, stress test concurrent document uploads

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
