# Завдання 35: Менеджер документів

## Предметна область

Система управління документами з папками, файлами та історією версій.

## Сутності

- **Folder**: Id, Name, ParentFolderId (nullable), CreatedBy, CreatedAt
- **Document**: Id, FolderId, Name, ContentType, SizeBytes, Version, CreatedAt, UpdatedAt
- **DocumentVersion**: Id, DocumentId, VersionNumber, ChangeSummary, CreatedBy, CreatedAt

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/folders | Отримати кореневі папки |
| GET | /api/folders/{id} | Отримати вміст папки (підпапки + документи) |
| POST | /api/folders | Створити папку |
| DELETE | /api/folders/{id} | Видалити папку (повинна бути порожньою) |
| POST | /api/documents | Завантажити документ |
| GET | /api/documents/{id} | Отримати метадані документа |
| PUT | /api/documents/{id} | Оновити документ (створює нову версію) |
| DELETE | /api/documents/{id} | Видалити документ |
| GET | /api/documents/{id}/versions | Отримати історію версій |
| GET | /api/documents/search?name={query} | Пошук документів за назвою |

## Бізнес-правила

- Назва папки повинна бути унікальною в межах батьківської папки
- Назва документа повинна бути унікальною в межах папки
- Не можна видаляти непорожні папки
- Кожне оновлення збільшує версію та створює запис версії
- Максимальний розмір файлу: 50 МБ
- Максимальна глибина вкладеності папок: 10 рівнів

## Обсяг тестування

- **Модульні тести**: Логіка збільшення версії, валідація глибини, перевірка обмеження розміру
- **Інтеграційні тести (WebApplicationFactory)**: Процес завантаження/оновлення, навігація по папках, історія версій, пошук
- **Тести бази даних (Testcontainers)**: Обмеження унікальності назв, рекурсивні запити до папок, цілісність ланцюжка версій
- **Тести продуктивності (k6)**: Навантажувальне тестування навігації по папках, стрес-тестування одночасного завантаження документів

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Folder**: Name, CreatedBy, CreatedAt
- **Document**: Name, ContentType, CreatedAt, UpdatedAt
- **DocumentVersion**: ChangeSummary, CreatedBy, CreatedAt

> Поля ParentFolderId, FolderId, SizeBytes, Version, VersionNumber мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу повинен відображати реальні співвідношення між батьківськими та дочірніми сутностями.
