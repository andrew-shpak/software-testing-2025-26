# Завдання 37: Відгуки на товари

## Предметна область

Система відгуків та оцінок товарів для електронної комерції з модерацією.

## Сутності

- **Product**: Id, Name, Description, Category, AverageRating, ReviewCount
- **Review**: Id, ProductId, ReviewerName, ReviewerEmail, Rating (1-5), Title, Content, CreatedAt, Status (Pending/Approved/Rejected), IsVerifiedPurchase
- **HelpfulVote**: Id, ReviewId, VoterEmail, IsHelpful

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/products | Отримати товари з оцінками |
| GET | /api/products/{id}/reviews | Отримати схвалені відгуки (сортування за корисністю/датою) |
| POST | /api/products/{id}/reviews | Надіслати відгук |
| PATCH | /api/reviews/{id}/approve | Схвалити відгук |
| PATCH | /api/reviews/{id}/reject | Відхилити відгук |
| POST | /api/reviews/{id}/helpful | Позначити відгук як корисний/некорисний |
| GET | /api/products/{id}/rating-breakdown | Отримати розподіл оцінок (кількість 1-5 зірок) |
| GET | /api/reviews/pending | Отримати відгуки, що очікують модерації |

## Бізнес-правила

- Один відгук на email на товар
- AverageRating та ReviewCount є денормалізованими (оновлюються при схваленні/відхиленні)
- Оцінка повинна бути від 1 до 5
- Тільки схвалені відгуки є видимими
- Відгуки підтверджених покупців виділяються
- Не можна голосувати за власний відгук

## Обсяг тестування

- **Модульні тести**: Розрахунок середньої оцінки, розподіл оцінок, правила модерації
- **Інтеграційні тести (WebApplicationFactory)**: Надсилання відгуку, процес модерації, голосування за корисність, розподіл оцінок
- **Тести бази даних (Testcontainers)**: Консистентність денормалізованої оцінки, унікальність відгуку, відстеження голосів за корисність
- **Тести продуктивності (k6)**: Навантажувальне тестування відгуків товарів з сортуванням, стрес-тестування одночасного надсилання відгуків

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Product**: Name, Description, Category
- **Review**: ReviewerName, ReviewerEmail, Title, Content, CreatedAt
- **HelpfulVote**: VoterEmail

> Поля AverageRating, ReviewCount, Rating, Status, IsVerifiedPurchase, IsHelpful, ProductId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу повинен відображати реальні співвідношення між батьківськими та дочірніми сутностями.
