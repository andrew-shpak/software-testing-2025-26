# Task 37: Product Review

## Domain

E-commerce product review and rating system with moderation.

## Entities

- **Product**: Id, Name, Description, Category, AverageRating, ReviewCount
- **Review**: Id, ProductId, ReviewerName, ReviewerEmail, Rating (1-5), Title, Content, CreatedAt, Status (Pending/Approved/Rejected), IsVerifiedPurchase
- **HelpfulVote**: Id, ReviewId, VoterEmail, IsHelpful

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/products | Get products with ratings |
| GET | /api/products/{id}/reviews | Get approved reviews (sort by helpful/recent) |
| POST | /api/products/{id}/reviews | Submit a review |
| PATCH | /api/reviews/{id}/approve | Approve a review |
| PATCH | /api/reviews/{id}/reject | Reject a review |
| POST | /api/reviews/{id}/helpful | Mark review as helpful/not helpful |
| GET | /api/products/{id}/rating-breakdown | Get rating distribution (1-5 star counts) |
| GET | /api/reviews/pending | Get reviews pending moderation |

## Business Rules

- One review per email per product
- AverageRating and ReviewCount are denormalized (updated on approve/reject)
- Rating must be 1-5
- Only approved reviews are visible
- Verified purchase reviews are highlighted
- Cannot vote on own review

## Testing Scope

- **Unit Tests**: Average rating calculation, rating distribution, moderation rules
- **Integration Tests (WebApplicationFactory)**: Review submission, moderation flow, helpful voting, rating breakdown
- **Database Tests (Testcontainers)**: Denormalized rating consistency, review uniqueness, helpful vote tracking
- **Performance Tests (k6)**: Load test product reviews with sorting, stress test concurrent review submissions

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Product**: Name, Description, Category
- **Review**: ReviewerName, ReviewerEmail, Title, Content, CreatedAt
- **HelpfulVote**: VoterEmail

> Fields like AverageRating, ReviewCount, Rating, Status, IsVerifiedPurchase, IsHelpful, ProductId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
