# Завдання 33: Конструктор опитувань

## Предметна область

Платформа для створення опитувань з питаннями, відповідями та аналітикою.

## Сутності

- **Survey**: Id, Title, Description, CreatedBy, CreatedAt, IsActive, ExpiresAt
- **Question**: Id, SurveyId, Text, Type (Text/SingleChoice/MultipleChoice/Rating), IsRequired, Order
- **Option**: Id, QuestionId, Text, Order
- **Response**: Id, SurveyId, RespondentEmail, SubmittedAt
- **Answer**: Id, ResponseId, QuestionId, Value

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/surveys | Отримати активні опитування |
| POST | /api/surveys | Створити опитування з питаннями |
| GET | /api/surveys/{id} | Отримати опитування з питаннями |
| PUT | /api/surveys/{id} | Оновити опитування |
| PATCH | /api/surveys/{id}/activate | Активувати/деактивувати |
| POST | /api/surveys/{id}/respond | Надіслати відповіді |
| GET | /api/surveys/{id}/results | Отримати агреговані результати |
| GET | /api/surveys/{id}/export | Експортувати відповіді у форматі JSON |

## Бізнес-правила

- На всі обов'язкові питання потрібно відповісти
- Відповіді SingleChoice повинні відповідати одному з варіантів
- Оцінка (Rating) повинна бути від 1 до 5
- Одна відповідь на email на одне опитування
- Не можна відповідати на неактивні/прострочені опитування
- Питання зберігають порядок в межах опитування

## Обсяг тестування

- **Модульні тести**: Валідація відповідей за типом питання, перевірка обов'язкових полів, межі оцінки
- **Інтеграційні тести (WebApplicationFactory)**: Створення опитування, надсилання відповідей, агрегація результатів
- **Тести бази даних (Testcontainers)**: Унікальність відповіді за email, порядок питань, каскадне видалення опитування
- **Тести продуктивності (k6)**: Навантажувальне тестування агрегації результатів, стрес-тестування одночасного надсилання відповідей

## Поля AutoFixture

Поля для автоматичної генерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Survey**: Title, Description, CreatedBy, CreatedAt
- **Question**: Text
- **Option**: Text
- **Response**: RespondentEmail, SubmittedAt

> Поля IsActive, ExpiresAt, Type, IsRequired, Order, Value, QuizId, QuestionId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як двигун бази даних. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Приклад розподілу повинен відображати реальні співвідношення між батьківськими та дочірніми сутностями.
