# Task 33: Survey Builder

## Domain

Survey creation platform with questions, responses, and analytics.

## Entities

- **Survey**: Id, Title, Description, CreatedBy, CreatedAt, IsActive, ExpiresAt
- **Question**: Id, SurveyId, Text, Type (Text/SingleChoice/MultipleChoice/Rating), IsRequired, Order
- **Option**: Id, QuestionId, Text, Order
- **Response**: Id, SurveyId, RespondentEmail, SubmittedAt
- **Answer**: Id, ResponseId, QuestionId, Value

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/surveys | Get active surveys |
| POST | /api/surveys | Create a survey with questions |
| GET | /api/surveys/{id} | Get survey with questions |
| PUT | /api/surveys/{id} | Update survey |
| PATCH | /api/surveys/{id}/activate | Activate/deactivate |
| POST | /api/surveys/{id}/respond | Submit responses |
| GET | /api/surveys/{id}/results | Get aggregated results |
| GET | /api/surveys/{id}/export | Export responses as JSON |

## Business Rules

- All required questions must be answered
- SingleChoice answers must match one of the options
- Rating must be 1-5
- One response per email per survey
- Cannot respond to inactive/expired surveys
- Questions maintain order within survey

## Testing Scope

- **Unit Tests**: Answer validation by question type, required field check, rating bounds
- **Integration Tests (WebApplicationFactory)**: Survey creation, response submission, result aggregation
- **Database Tests (Testcontainers)**: Response uniqueness per email, question ordering, cascading survey delete
- **Performance Tests (k6)**: Load test result aggregation, stress test concurrent response submissions

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Survey**: Title, Description, CreatedBy, CreatedAt
- **Question**: Text
- **Option**: Text
- **Response**: RespondentEmail, SubmittedAt

> Fields like IsActive, ExpiresAt, Type, IsRequired, Order, Value, QuizId, QuestionId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
