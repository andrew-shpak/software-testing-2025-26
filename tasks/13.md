# Task 13: Student Gradebook

## Domain

Academic gradebook for managing students, courses, and grades.

## Entities

- **Student**: Id, FirstName, LastName, Email, StudentNumber, EnrollmentDate
- **Course**: Id, Name, Code, Credits, MaxStudents
- **Enrollment**: Id, StudentId, CourseId, Grade (nullable), EnrolledAt
- **GradeRecord**: Id, EnrollmentId, AssignmentName, Score, MaxScore, GradedAt

## Endpoints

| Method | Route | Description |
|--------|-------|-------------|
| GET | /api/students | Get all students |
| POST | /api/students | Register a student |
| GET | /api/courses | Get all courses |
| POST | /api/courses | Create a course |
| POST | /api/courses/{id}/enroll | Enroll a student |
| POST | /api/enrollments/{id}/grades | Add a grade record |
| GET | /api/students/{id}/transcript | Get student transcript (all courses + grades) |
| GET | /api/courses/{id}/grades | Get all grades for a course |

## Business Rules

- Student cannot enroll in the same course twice
- Cannot exceed MaxStudents for a course
- Score cannot exceed MaxScore
- Final grade is calculated as weighted average of all grade records
- StudentNumber must be unique

## Testing Scope

- **Unit Tests**: Grade calculation (weighted average), enrollment validation, score bounds
- **Integration Tests (WebApplicationFactory)**: Enrollment flow, grade submission, transcript generation
- **Database Tests (Testcontainers)**: Unique enrollment constraint, max students enforcement, grade aggregation queries
- **Performance Tests (k6)**: Load test transcript generation, stress test concurrent enrollments

## AutoFixture Fields

Fields to auto-generate with AutoFixture (not business-critical, just need valid data):

- **Student**: FirstName, LastName, Email, EnrollmentDate
- **Course**: Name, Description
- **GradeRecord**: AssignmentName, GradedAt

> Fields like StudentNumber, Code, Credits, MaxStudents, Grade, Score, MaxScore, StudentId, CourseId have business rules and should be set explicitly in tests.

## Database

Use **PostgreSQL** as the database engine. For Testcontainers, use `Testcontainers.PostgreSql` package.

## Database Seeding

For performance and integration tests, the database must be pre-seeded with **at least 10,000 records** distributed across all entities. Use AutoFixture (or Bogus) to generate realistic test data. Example distribution should reflect real-world ratios between parent and child entities.
