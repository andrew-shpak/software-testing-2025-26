# Завдання 13: Журнал оцінок студентів

## Домен

Академічний журнал оцінок для управління студентами, курсами та оцінками.

## Сутності

- **Student**: Id, FirstName, LastName, Email, StudentNumber, EnrollmentDate
- **Course**: Id, Name, Code, Credits, MaxStudents
- **Enrollment**: Id, StudentId, CourseId, Grade (nullable), EnrolledAt
- **GradeRecord**: Id, EnrollmentId, AssignmentName, Score, MaxScore, GradedAt

## Ендпоінти

| Метод | Маршрут | Опис |
|--------|-------|-------------|
| GET | /api/students | Отримати всіх студентів |
| POST | /api/students | Зареєструвати студента |
| GET | /api/courses | Отримати всі курси |
| POST | /api/courses | Створити курс |
| POST | /api/courses/{id}/enroll | Зарахувати студента |
| POST | /api/enrollments/{id}/grades | Додати запис оцінки |
| GET | /api/students/{id}/transcript | Отримати транскрипт студента (всі курси + оцінки) |
| GET | /api/courses/{id}/grades | Отримати всі оцінки за курс |

## Бізнес-правила

- Студент не може бути зарахований на один і той самий курс двічі
- Не можна перевищити MaxStudents для курсу
- Score не може перевищувати MaxScore
- Підсумкова оцінка розраховується як зважене середнє всіх записів оцінок
- StudentNumber має бути унікальним

## Обсяг тестування

- **Юніт-тести**: Розрахунок оцінки (зважене середнє), валідація зарахування, межі балів
- **Інтеграційні тести (WebApplicationFactory)**: Процес зарахування, подання оцінок, генерація транскрипту
- **Тести бази даних (Testcontainers)**: Обмеження унікальності зарахування, контроль максимальної кількості студентів, запити агрегації оцінок
- **Тести продуктивності (k6)**: Навантажувальне тестування генерації транскрипту, стрес-тестування одночасних зарахувань

## Поля AutoFixture

Поля для автогенерації за допомогою AutoFixture (не критичні для бізнес-логіки, потрібні лише валідні дані):

- **Student**: FirstName, LastName, Email, EnrollmentDate
- **Course**: Name, Description
- **GradeRecord**: AssignmentName, GradedAt

> Поля StudentNumber, Code, Credits, MaxStudents, Grade, Score, MaxScore, StudentId, CourseId мають бізнес-правила і повинні встановлюватися явно в тестах.

## База даних

Використовуйте **PostgreSQL** як СУБД. Для Testcontainers використовуйте пакет `Testcontainers.PostgreSql`.

## Наповнення бази даних

Для тестів продуктивності та інтеграційних тестів база даних повинна бути попередньо наповнена **щонайменше 10 000 записами**, розподіленими між усіма сутностями. Використовуйте AutoFixture (або Bogus) для генерації реалістичних тестових даних. Розподіл має відображати реальні співвідношення між батьківськими та дочірніми сутностями.
